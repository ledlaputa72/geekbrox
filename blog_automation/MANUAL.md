## 1. 빠른 시작

블로그 글 1편을 만들고 게시하는 가장 빠른 경로:

```
① 텔레그램에서 /start 입력
② 1️⃣ 블로그 제작 버튼 클릭
③ 1-1 🔍 자료조사 버튼 → 완료 알림 대기 (30초~1분)
④ 1-2 ✍️ 글 생성 버튼 → 완료 알림 대기 (1~3분)
⑤ 1-3 📋 초안 확인 버튼 → 내용 확인
⑥ 1-4 🚀 포스팅 실행 버튼 → Tistory 게시 완료
```

> 💡 **모든 작업은 버튼 클릭만으로 완결됩니다.** 텍스트 입력은 초안 수정 지시나 메모에만 필요합니다.

## 2. 봇 실행 방법

### 사전 준비

```bash
# 의존성 설치
pip install python-telegram-bot>=20.0 python-dotenv

# .env 파일 설정 (프로젝트 루트)
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_CHAT_ID=your_chat_id_here
```

### 봇 시작

```bash
# 프로젝트 루트에서 실행
python blog_automation/scripts/content_team_bot.py
```

### 봇 종료

터미널에서 `Ctrl+C` 입력

## 3. 계층식 버튼 메뉴 구조

텔레그램 인라인 버튼으로 3단계 계층 구조를 탐색합니다.

```
🏠 홈
├── 1️⃣  블로그 제작
│   ├── 1-1  🔍 자료조사
│   ├── 1-2  ✍️ 글 생성
│   │         └── [확인] 버튼으로 실행 확정
│   ├── 1-3  📋 초안 확인 (N개)
│   │         ├── 📄 [제목] → 내용 보기
│   │         ├── ✏️ 수정  → 수정 지시문 입력 후 Claude 재작성
│   │         └── 🗑 삭제
│   └── 1-4  🚀 포스팅 실행
│             └── [확인] 버튼으로 실행 확정
│
├── 2️⃣  현황 & 통계
│   ├── 2-1  ⚙️ 블로그 현황
│   ├── 2-2  📰 게시 완료 목록
│   └── 2-3  📊 API & 큐 상태
│
├── 3️⃣  3-Way 공유
│   ├── 3-1  🔗 공유 현황
│   ├── 3-2  📋 활동 로그
│   ├── 3-3  🚨 충돌 확인
│   │         └── ✅ 충돌 해제 버튼
│   └── 3-4  💬 메시지 전달
│             ├── 🖥 Claude Code에게
│             └── 🎯 Cursor AI에게
│
└── 4️⃣  도움말
    ├── 4-1  🚀 빠른 시작
    ├── 4-2  🔲 버튼 메뉴 안내
    ├── 4-3  🔘 슬래시 명령어
    ├── 4-4  💡 팁 & 설정
    └── 📋 전체 명령어 보기
```

모든 하위 메뉴에는 **🏠 홈** 버튼이 있어 언제든지 홈으로 돌아올 수 있습니다.

## 4. 각 메뉴 상세 설명

### 1️⃣ 블로그 제작

| 버튼 | 실행 스크립트 | 설명 |
|------|--------------|------|
| **1-1 🔍 자료조사** | `fetch_anime.py` | AniList API에서 이번 시즌 애니 Top 10 데이터 수집. 결과: `teams/content/workspace/blog/data/seasonal_top_anime.json` |
| **1-2 ✍️ 글 생성** | `generate_post.py` | 수집된 데이터로 블로그 글 초안 생성. Claude/Gemini LLM 사용. 결과: `teams/content/workspace/blog/drafts/` |
| **1-3 📋 초안 확인** | — | 생성된 초안 목록 표시 (최대 8개). 각 초안마다 보기/수정/삭제 버튼 |
| **1-4 🚀 포스팅 실행** | `post_to_tistory.py` | 초안을 Tistory에 자동 게시. 완료 후 `published/` 폴더로 이동 |

#### 초안 수정 방법

1. **1-3 초안 확인** → 해당 초안의 **✏️ 수정** 버튼 클릭
2. 텔레그램에 수정 지시문 입력 (예: `도입부를 더 흥미롭게 바꿔줘`)
3. Claude가 지시에 따라 초안을 자동으로 재작성

### 2️⃣ 현황 & 통계

| 버튼 | 표시 내용 |
|------|---------|
| **2-1 ⚙️ 블로그 현황** | 초안 대기 수, 게시 완료 수, 이미지 수, API 안전 여부 |
| **2-2 📰 게시 완료 목록** | 최근 게시된 글 최대 20개 목록 |
| **2-3 📊 API & 큐 상태** | 60초/5초 API 호출 횟수, Rate Limit 위험도, 대기 큐 상태 |

### 3️⃣ 3-Way 공유

Claude Code ↔ Cursor AI ↔ 텔레그램 봇 3자 간 실시간 상태 공유.

| 버튼 | 표시 내용 |
|------|---------|
| **3-1 🔗 공유 현황** | Claude Code 현재 작업, Cursor AI 현재 작업, 마지막 업데이트 시각 |
| **3-2 📋 활동 로그** | 최근 15개 작업 내역 (누가, 언제, 무엇을) |
| **3-3 🚨 충돌 확인** | 동시 편집 충돌 목록. 심각도: CRITICAL(즉시 해결) / WARNING(주의) |
| **3-4 💬 메시지 전달** | Claude Code 또는 Cursor AI에게 텍스트 메시지 전달 |

#### 메시지 전달 방법

1. **3-4 💬 메시지 전달** → 대상 선택 (`Claude Code` 또는 `Cursor AI`)
2. 텔레그램에 전달할 메시지 입력
3. 메시지가 `shared_state.json`에 저장되어 다음 작업 시작 시 해당 AI가 확인

### 4️⃣ 도움말

봇 내에서 바로 확인할 수 있는 도움말 서브메뉴.

| 버튼 | 내용 |
|------|------|
| **4-1 🚀 빠른 시작** | 블로그 1편 제작 6단계 가이드 |
| **4-2 🔲 버튼 메뉴 안내** | 전체 버튼 계층 구조 트리 |
| **4-3 🔘 슬래시 명령어** | 사용 가능한 `/` 명령어 목록 |
| **4-4 💡 팁 & 설정** | 소요 시간, 자동 알림, `.env` 환경변수 안내 |

## 5. 텍스트 입력 명령어 (최소 사용)

> ⚠️ **봇은 버튼 제어 중심으로 설계되어 있습니다.** 아래 4가지 텍스트 외 일반 대화는 응답하지 않습니다.

| 텍스트 형식 | 기능 |
|------------|------|
| `메모: [내용]` | Claude Code에 메모 전달 (shared_state 기록) |
| `note: [내용]` | `메모:` 와 동일 (영문) |
| `인증완료` | 카카오 추가 인증 완료 알림 전달 |
| **수정 지시문** | 초안 ✏️ 수정 버튼 클릭 후, 다음 메시지로 수정 내용 입력 |

### 텍스트 입력 예시

```
메모: 오늘 3편 예약 포스팅 설정 완료

note: generate_post.py timeout 늘려야 함

인증완료

(✏️ 수정 버튼 클릭 후)
결말 부분을 좀 더 감성적으로 다듬어줘. SEO 키워드는 '2025 애니 추천' 유지.
```

## 7. 3-Way 공유 시스템

```
┌─────────────────────────────────────────────┐
│            shared_state.json                │
│   teams/content/workspace/shared_state.json │
├──────────────┬──────────────┬───────────────┤
│  Claude Code │  Cursor AI   │ Telegram Bot  │
│  (자동 쓰기) │  (자동 쓰기) │  (읽기 전용)  │
└──────────────┴──────────────┴───────────────┘
```

### 공유 상태 파일 위치

| 파일 | 역할 |
|------|------|
| `teams/content/workspace/shared_state.json` | 현재 작업자, 상태, 메시지 큐 |
| `teams/content/workspace/activity_log.json` | 전체 작업 내역 로그 |
| `teams/content/workspace/conflicts.json` | 충돌 기록 및 해제 이력 |

### 충돌 심각도

| 심각도 | 의미 | 대응 |
|--------|------|------|
| `CRITICAL` | 동일 파일 동시 편집 등 즉각 해결 필요 | 작업 중단 후 **3-3 충돌 해제** |
| `WARNING` | 주의가 필요하지만 작업 계속 가능 | 확인 후 **3-3 충돌 해제** |

## 8. AI 크레딧 절약 팁

봇은 **버튼 콜백 방식**으로 설계되어 버튼 클릭 자체는 LLM 호출이 없습니다.

| 동작 | AI 호출 여부 |
|------|------------|
| 버튼 클릭 (메뉴 이동, 현황 조회) | ❌ 없음 |
| 자료조사 (`fetch_anime.py`) | ❌ 없음 (AniList API) |
| 글 생성 (`generate_post.py`) | ✅ Claude/Gemini 호출 |
| 초안 수정 지시 | ✅ Claude 호출 |
| 포스팅 (`post_to_tistory.py`) | ❌ 없음 (Selenium) |

### Rate Limit 방지 설계

- **글 간 딜레이**: 기본 30초 (`INTER_POST_DELAY` 환경변수로 조절)
- **자동 큐 처리**: 여러 글 생성 시 순차 처리 (동시 호출 방지)
- **Exponential Backoff**: 실패 시 자동 재시도 (2초 → 4초 → 8초...)
- **API 호출 추적**: 60초/5초 구간 호출 횟수 모니터링 → **2-3 API 상태** 에서 확인

## 9. 환경변수 설정 (.env)

프로젝트 루트에 `.env` 파일 생성:

```env
# 텔레그램 봇 필수 설정
TELEGRAM_BOT_TOKEN=1234567890:ABCdef...    # BotFather에서 발급
TELEGRAM_CHAT_ID=987654321                 # 본인 chat_id (선택)

# Rate Limit 설정
INTER_POST_DELAY=30          # 글 간 대기 시간(초), 기본 30
LLM_MAX_RETRY=4              # LLM 실패 시 재시도 횟수, 기본 4

# LLM API (generate_post.py 에서 사용)
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=AIza...

# Tistory (post_to_tistory.py 에서 사용)
TISTORY_ACCESS_TOKEN=...
TISTORY_BLOG_NAME=geekbrox
```

> 💡 `TELEGRAM_CHAT_ID` 를 설정하면 지정된 사용자만 봇을 사용할 수 있습니다 (보안 권장).

## 11. 자주 묻는 질문 (FAQ)

**Q. 텍스트를 보냈는데 봇이 응답을 안 해요.**
A. 정상 동작입니다. 봇은 버튼 제어 중심으로 설계되어 인식하지 못하는 텍스트는 무시합니다. `/start` 로 홈 메뉴를 열어 버튼을 사용하세요.

---

**Q. 초안이 생성됐는데 1-3 초안 확인에서 안 보여요.**
A. 초안 파일이 `teams/content/workspace/blog/drafts/` 폴더에 `.md` 형식으로 저장되어 있는지 확인하세요. 다른 경로에 저장된 파일은 표시되지 않습니다.

---

**Q. 글 생성이 너무 오래 걸려요.**
A. `generate_post.py` 의 LLM API 호출 시간에 따라 다릅니다. 5분 초과 시 타임아웃으로 자동 종료됩니다. Rate Limit 상황이면 **2-3 API 상태** 에서 확인 후 대기하세요.

---

**Q. Tistory 포스팅에서 카카오 추가 인증이 요구돼요.**
A. Selenium 자동화 중 카카오 2차 인증이 뜨면 수동으로 인증 후 텔레그램에 `인증완료` 를 입력하면 봇이 인증 완료를 감지하고 계속 진행합니다.

---

**Q. 충돌(CRITICAL)이 발생했어요.**
A. Claude Code와 Cursor AI가 동시에 같은 파일을 편집하면 충돌이 발생합니다. **3-3 충돌 확인** → **✅ 충돌 해제** 버튼을 눌러 해제하세요.

---

**Q. 봇을 서버에서 상시 실행하려면?**
A. `pm2`, `screen`, `tmux`, 또는 `systemd` 서비스로 등록하세요.
```bash
# pm2 예시
pm2 start "python blog_automation/scripts/content_team_bot.py" --name geekbrox-bot
pm2 save
```
