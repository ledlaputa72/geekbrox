## 📌 Executive Summary (핵심 요약)

세 게임 장르의 전투 시스템을 분석한 결과, Dream Collector가 지향해야 할 전투 시스템의 방향이 명확해졌다.

> **"StS의 전략적 덱빌딩 + Expedition 33의 실시간 반응 + 소울류의 학습 쾌감"**
>
> 이 세 요소의 융합이 Dream Collector를 단순한 모바일 카드 게임이 아닌,
> **"모바일에서 가장 긴장감 있는 카드 전투"** 로 만드는 열쇠다.

**핵심 결론**: 현재 Dream Collector의 전투 시스템(ATB v3.0 + 턴베이스 v1.0 + 리액션 v3.0)은 올바른 방향으로 설계되고 있음. 이 리포트는 세 장르의 **핵심 DNA를 검증**하고, 추가 적용 우선순위를 제시한다.

## 2. 각 장르별 Dream Collector 기여 분석

### 2.1 StS → Dream Collector 기여

StS는 Dream Collector의 **뼈대(Skeleton)**:

```
StS에서 직접 가져온 것:
✅ 에너지 시스템 (매 턴 3 충전)
✅ 손패 드로우 (5장/턴)
✅ 블록 시스템 (턴 종료 시 소멸)
✅ 의도(Intent) 시스템 (적 행동 예고)
✅ 덱빌딩 로그라이크 루프
✅ 카드 유형 분류 (공격/방어/스킬)
✅ 상태이상 시스템 (중독/취약/약화 등)

StS에서 개선한 것:
🔼 의도 시스템 → 3단계 연출 + 다음 2~3행동 예고
🔼 전투 중 수동성 → 리액션 시스템으로 제거
🔼 단순한 블록 → 완전 패링/반응 카드/긴급 가드로 다층화
```

**StS의 최대 약점 보완**: "적 턴에 아무것도 안 한다" → 리액션 시스템으로 완전 해소.

### 2.2 Expedition 33 → Dream Collector 기여

Expedition 33은 Dream Collector의 **심장(Heart)**:

```
Expedition 33에서 가져온 것:
✅ 패링 = 리스크/리워드 (완전 패링 → 반격 카드)
✅ 체인 패링 → 체인 카운터 보너스
✅ 브레이크 게이지 → 자세 게이지 시스템
✅ 그라디언트 카운터 → 보스 전용 특수 연출
✅ 시청각 예고 큐 → 3단계 공격 예고 연출
✅ 접근성 옵션 → Story 모드 윈도우 확장

Dream Collector의 적응 변형:
🔄 실시간 패링 버튼 → 카드 탭 타이밍
🔄 3인칭 뷰 → 2D 카드 UI
🔄 팀 원정대 카운터 → 꿈의 방어막 카드
🔄 QTE 스킬 강화 → 패링 성공 시 반격 카드 강화
```

### 2.3 소울류 → Dream Collector 기여

소울류는 Dream Collector의 **깊이(Depth)**:

```
소울류에서 가져온 것:
✅ 자세 게이지 (Sekiro) → 몬스터 자세 파쇄 시스템
✅ 공격 예고(Tell) 철학 → 모든 공격에 예고 의무화
✅ 危 표시 (관통 공격) → "방어 불가!" 표시
✅ 패턴 학습 보상 → 반복 처치 시 예고 더 일찍 표시
✅ 학습 → 성취 사이클 → 패링 마스터 시 전투 효율 극대화
✅ 페이크 공격 → 보스 전용 페이크 패턴

모바일 변환:
🔄 실시간 자세 관리 → 턴제/ATB 구조 내 누적
🔄 스태미나 → 에너지 시스템이 대체
🔄 Controller 패링 → 터치 탭 타이밍
```

## 3. Dream Collector만의 독자 포지션

세 장르의 요소를 가져왔지만, Dream Collector만의 독자성도 중요:

### 3.1 Dream Collector의 차별화 포인트

```
Dream Collector = StS × Expedition 33 × 소울류 + 독자 요소

독자 요소:
🌙 타로 카드 테마 (78장 아르카나 시스템)
🌙 꿈/악몽 세계관 설정
🌙 ATB + 턴제 선택 가능 (두 전투 모드)
🌙 모바일 완전 최적화 (오토 플레이 3단계)
🌙 꿈 보조자 루미 (초보자 가이드 캐릭터)
🌙 전투 일지 시스템 (학습 피드백)
🌙 꿈 조각 시스템 (독자 자원)
```

### 3.2 세계 최초 조합

업계에서 아직 없는 조합:

```
[아직 없는 조합]
모바일 최적화 + 패링 타이밍 + 덱빌딩 로그라이크

현존 유사작:
- StS 모바일: 패링 없음, 실시간 요소 없음
- Expedition 33: 모바일 없음, 덱빌딩 없음
- 소울류 모바일: 덱빌딩 없음

Dream Collector: 세 요소 모두 통합 → 블루오션 포지션
```

## 7. 최종 권장사항

### OPS 팀의 핵심 권장 3가지

**권장 1. "패링 우선 구현"**
완전 패링 시스템은 Dream Collector의 가장 강력한 차별화 요소. 구현 품질이 게임 전체 평가를 결정. 반드시 2단계에서 충분한 일정과 자원 배정 필요.

**권장 2. "접근성과 깊이의 이중 구조 유지"**
StS의 깊이 + Expedition 33의 실시간 요소는 하드코어 유저에게 매력적. 하지만 오토 플레이 + 루미 + Story 모드 확장 윈도우 없이는 캐주얼 유저 이탈 발생. 두 레이어를 동시에 완성해야 함.

**권장 3. "2차 OPS 테스트 — 패링 중심"**
리액션 시스템 v3.0 구현 후 반드시 OPS 팀 시뮬레이션 테스트 재실시. 특히 **모바일에서 0.4초 패링 윈도우의 체감**과 **자세 게이지 UI의 직관성**이 핵심 검증 항목.
