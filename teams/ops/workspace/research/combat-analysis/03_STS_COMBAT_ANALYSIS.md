## 2. 핵심 전투 구조

### 2.1 기본 턴 흐름

```
[턴 시작]
├── 에너지 3 충전
├── 카드 드로우 5장
├── 몬스터 의도(Intent) 표시

[플레이어 턴]
├── 에너지를 소모하며 카드 플레이 (순서 자유)
├── 효과 즉시 적용
└── 원하는 때 "턴 종료" 가능

[플레이어 턴 종료]
├── 사용한 카드 → 버림 더미
├── 사용 안 한 카드 → 버림 더미 (기본값)
└── 블록 소멸

[몬스터 턴]
├── 예고된 의도대로 행동
└── 상태이상 적용

→ 플레이어 턴 시작으로 루프
```

### 2.2 에너지 시스템

```
에너지: 매 턴 고정값(기본 3) 충전 → 카드 비용에 소모
특징:
├── 턴 끝나면 남은 에너지 소멸 (이월 없음)
├── 카드마다 에너지 비용 다름 (0~5)
└── 고비용 = 강한 효과 / 저비용 = 약하지만 여러 장 사용
```

**디자인 의미**: 에너지는 **"무엇을 할 것인가"의 우선순위 선택** 강제. 모든 것을 다 할 수 없기 때문에 전략적 선택이 발생.

### 2.3 카드 유형

| 유형 | 역할 | 대표 효과 |
|------|------|---------|
| Attack | 피해 | 데미지, 상태이상 부여 |
| Skill | 유틸 | 블록 생성, 드로우, 버프 |
| Power | 지속 효과 | 매 턴 적용되는 패시브 |
| Curse | 핸디캡 | 방해 효과 (주로 적이 부여) |

**특이점**: Power 카드는 사용 후 덱에 돌아가지 않고 전투 내내 효과 지속.

## 3. 의도(Intent) 시스템 — StS의 핵심 혁신

### 3.1 개념

StS의 가장 독창적인 시스템 중 하나. 적의 **다음 턴 행동을 미리 공개**:

```
[적 HP 바 위 의도 표시]
꿈의 악몽
HP: ████████░░ 82%
[⚔️ 15]  ← 다음 턴에 15 데미지 공격 예정
```

### 3.2 의도 아이콘 체계

| 아이콘 | 의미 | 플레이어 대응 |
|--------|------|------------|
| ⚔️ (숫자) | 공격 + 예상 데미지 | 블록 준비 |
| 🛡️ | 방어 강화 | 이번 턴 공격 집중 or 디버프 |
| ✨ | 버프 획득 | 버프 제거 카드 준비 |
| ☠️ | 강화 공격 (위험) | 최대 블록 필요 |
| 🔀 | 패턴 섞기 | 다음 턴 예측 불가 |
| 💤 | 이번 턴 쉼 | 공격 집중 타이밍 |

### 3.3 의도 시스템의 게임 디자인적 의의

의도 공개는 처음엔 "너무 쉬운 거 아닌가?"처럼 보이지만:

```
의도를 알아도 어려운 이유:
├── 에너지가 제한되어 있어 블록 + 공격 모두 불가
├── 손패에 원하는 카드가 없을 수 있음
├── 여러 적의 의도를 동시에 처리해야 함
└── 복수 적의 누적 데미지 계산 필요
```

**결론**: 의도 공개 = 정보 제공, 하지만 그 정보를 **실현하는 것은 여전히 어려움**. 정보와 실행의 분리가 전략 게임의 깊이를 만듦.

### 3.4 의도 패턴의 설계

좋은 몬스터 설계 = 예측 가능하면서도 대응이 어려운 패턴:

```
[패턴 예시: 방어형 적]
턴 1: 방어 강화 (5 블록)
턴 2: 강한 공격 (20 데미지)
턴 3: 약한 공격 + 디버프
→ 반복

플레이어 선택 강제:
└── 턴 1에 공격해서 블록 전에 데미지 넣을 것인가?
└── 아니면 턴 1에 블록 쌓아서 턴 2를 버틸 것인가?
```

## 4. 덱빌딩 시스템 — 전투 외 전략 레이어

### 4.1 카드 획득 사이클

```
전투 승리 → 카드 3장 중 1장 선택 (or 거절)
상점 → 카드 구매/제거
캠프파이어 → 카드 업그레이드 (강화) or 휴식 (HP 회복)
```

**핵심 설계**: 카드 추가만큼 카드 **제거**도 중요. 덱이 너무 크면 원하는 카드를 드로우할 확률이 낮아짐.

```
최적 덱 크기: 약 12~20장
너무 작으면: 모든 상황 대응 불가
너무 크면: 핵심 콤보 카드 드로우 확률 하락
```

### 4.2 카드 시너지 (Synergy)

StS의 깊이의 원천. 개별 카드는 평범해도 조합하면 강력:

```
[시너지 예시]
"상처 입히기" (취약 부여) + "폭발" (취약 시 2배 데미지)
= 2배 데미지 콤보

"힘 강화" 카드 × 여러 장 = 공격력 지수 증가

"카드 드로우" 카드 × "0코스트 공격" 카드 = 무한 루프 콤보 가능
```

### 4.3 유물(Relic) 시스템

카드와 별개로 **수동 패시브 효과**를 부여하는 유물:

```
[유물 예시]
"칼날 공" — 전투 시작 시 칼날 5 생성 (방어+공격 동시)
"Akabeko" — 첫 번째 공격에 8 추가 데미지
"타르포스" — 전투마다 HP 1 회복
```

유물이 덱 전략의 방향을 결정하는 경우가 많음.

## 5. 상태이상 시스템

### 5.1 핵심 상태이상

| 상태이상 | 효과 | 해제 조건 |
|---------|------|---------|
| 중독(Poison) | 매 턴 HP 감소 (스택 1씩 감소) | 자연 소멸 |
| 취약(Vulnerable) | 받는 피해 50% 증가 | 2턴 후 |
| 약화(Weak) | 주는 피해 25% 감소 | 2턴 후 |
| 묶음(Entangled) | 이번 턴 공격 카드 사용 불가 | 1턴 후 |
| 힘(Strength) | 공격 피해 증가 | 영구 |
| 민첩(Dexterity) | 블록 획득량 증가 | 영구 |

### 5.2 상태이상의 전략적 역할

상태이상은 단순한 디버프가 아닌 **게임 내 경제(Resource Economy)**:

```
중독 전략: 중독 부여 카드 집중 투자 → 적이 알아서 죽음
취약 전략: 취약 부여 → 다음 공격 1.5배 → 더 적은 에너지로 같은 결과
힘 스택: 힘 부여 카드 × 다수 → 공격력 기하급수적 성장
```

## 6. 블록 시스템

### 6.1 블록의 특성

```
블록:
├── 방어 카드로 생성 (민첩 스탯에 비례)
├── 피해 대신 흡수
├── 매 턴 종료 시 전부 소멸 (기본값)
└── 일부 유물/카드로 이월 가능
```

**핵심 규칙**: 블록 이월 불가로 인해 매 턴 방어/공격 판단을 새로 해야 함.

### 6.2 블록 = 복잡한 전략 자원

```
상황별 블록 전략:
├── 적 공격 데미지 < 블록 가능량 → 완전 방어
├── 적 공격 데미지 > 블록 가능량 → 부분 방어 + 상태이상 관리
├── 적이 방어하는 턴 → 블록 쌓을 기회 or 방어 뚫기 카드
└── 적 죽음 직전 → 블록 무시하고 공격 집중
```

## 7. StS 전투의 핵심 재미 요소

### 7.1 "손패 계획" 의 재미

매 턴 드로우한 5장으로 이번 턴을 계획하는 퍼즐:

```
[이번 턴 손패]
공격(1) + 공격(2) + 방어(1) + 드로우(0) + 버프(1)

옵션:
A. 드로우로 추가 카드 → 더 나은 조합 기대
B. 버프 먼저 → 이후 공격 강화
C. 공격 집중 → 적 빠르게 처리
D. 방어 우선 → 이번 턴 생존 보장
```

이 계획 수립이 StS 전투의 핵심 인지적 재미.

### 7.2 "덱 콤보 발견" 의 재미

로그라이크 특성상 매 런 다른 카드 조합:
```
런 중 발견: "이 두 카드가 이렇게 강력했어?"
→ 발견의 즐거움 → 다음 런에서 의도적으로 시도
→ 또 다른 조합 발견 → 무한 탐험
```

### 7.3 리플레이성의 원천

```
매 런 다른 요소:
├── 랜덤 카드 제공 (전략 방향 유동적)
├── 랜덤 유물 제공
├── 랜덤 적 배치
└── 플레이어의 성장 (더 좋은 선택)
= 500시간 플레이해도 신선한 경험
```

## 8. StS의 한계 및 단점

| 한계 | 설명 |
|------|------|
| **실시간 긴장감 없음** | 모든 행동이 정적, 턴 사이 긴장감 부재 |
| **캐주얼 진입 장벽** | 수십 가지 카드/효과 학습 필요 |
| **시각적 단조로움** | 전투 연출이 단순 |
| **빈 시간** | 적의 행동을 "그냥 맞음" = 수동적 순간 |
| **모바일 UX 한계** | 작은 카드, 많은 정보 = 모바일에서 불편 |

## 10. Dream Collector 적용 가능성 평가

| StS 메카닉 | 적용 가능성 | 이미 반영 여부 |
|-----------|-----------|-------------|
| 에너지 시스템 | ★★★★★ 최고 | ✅ 이미 적용 (에너지 3) |
| 의도(Intent) 시스템 | ★★★★★ 최고 | ✅ v2.0~v3.0에 적용 |
| 블록 시스템 | ★★★★★ 최고 | ✅ 이미 적용 |
| 상태이상 시스템 | ★★★★☆ 높음 | ✅ 부분 적용 |
| 덱빌딩 시스템 | ★★★★★ 최고 | ✅ 이미 핵심 기능 |
| 카드 제거 시스템 | ★★★★☆ 높음 | ⚠️ 추후 적용 검토 |
| Power 카드 유형 | ★★★★☆ 높음 | ⚠️ 적용 검토 필요 |

**한계 극복 방향**: StS의 약점인 "실시간 긴장감 없음" + "적 턴의 수동성"을
Expedition 33 패러다임으로 보완 → Dream Collector의 차별점이 됨.
