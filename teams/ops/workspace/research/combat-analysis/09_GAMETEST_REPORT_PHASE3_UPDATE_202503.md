## 🎯 Cursor UI 개선사항 검증

### ✅ 1. 카드 비행 애니메이션

**구현 상태**: ✅ 완료 및 검증
**표시 내용**: 카드가 대상(플레이어/몬스터)으로 날아감 → 축소하며 사라짐

| 구간 | 시간 | 설명 | 체감 평가 |
|------|------|------|---------|
| 1단계 | 0.2s | 선택 카드 위로 올라가며 투명 | ✅ 자연스러움 |
| 2단계 | 0.0s | 전투 화면 중앙 재등장 | ✅ 명확한 카드 인식 |
| 3단계 | 0.35s | 대상에게 날아감 (축소+사라짐) | ✅ 만족도 높음 |
| **합계** | **0.55s** | **전체 애니** | ✅ 너무 길지 않음 |

**UX**: 사용자가 카드가 실제로 대상에게 적용되는 과정을 시각적으로 추적 가능. 게임성 향상.

**문제**: ❌ 애니메이션은 정상이나, **애니메이션 종료 후 `manager.player_play_card()` 호출 시점에 데미지가 적용되지 않음** (별도 조사 필요)

### ✅ 2. 효과 숫자 표시 (데미지·회복·블록)

**구현 상태**: ✅ 완료 및 부분 검증

| 효과 유형 | 색상 | 아이콘 | 구현 상태 |
|---------|------|--------|---------|
| **데미지** | 🔴 빨강 | `-숫자` | ✅ 정상 |
| **회복** | 🟢 녹색 | `+숫자` | ✅ 정상 |
| **블록** | 🔵 파랑 | `🛡+숫자` | ✅ 정상 |

**표시 위치**: 플레이어/몬스터 머리 위, 위로 떠오르며 페이드아웃
**발생 시점**: `damage_dealt`, `player_hp_changed` 신호 기반

**체감 평가**: ⭐⭐⭐⭐ (4/5)
- 플레이어가 즉시 효과를 인지 가능
- 전투 상황 이해도 증가
- 단점: 모바일 화면에 겹칠 수 있음 (2명 이상 데미지 시)

### ✅ 3. 액션 턴 큐 (상단 바)

**구현 상태**: ✅ 완료 및 검증
**위치**: 전투 화면 바로 위, 노란 박스
**표시 항목**: 최대 5개 턴, 왼쪽=현재(▲), 오른쪽=다음

| 항목 | 스펙 | 체감 |
|------|------|------|
| **높이** | 기존 40% | ✅ 적절 |
| **아이콘 크기** | 50% 축소 (15×15px) | ✅ 가독성 유지 |
| **삼각형 표시** | ▲ 흰색 | ✅ 명확함 |
| **현재 턴 테두리** | 2px 흰색 | ✅ 강조됨 |

**기능 검증**:
- ATB 전투: SPD 기반 예측 순서 ✅ 정상
- 턴베이스: 플레이어→적 순서 ✅ 정상
- 턴 전환 애니: 아이콘 이동 + 스케일 ✅ 부드러움

**장점**: 
- 플레이어가 다음 누가 움직일지 즉시 판단 가능
- 전략 깊이 증가 (미리 생각할 시간 제공)

---

### ✅ 4. 현재 액션 대상 표시

**구현 상태**: ✅ 완료 및 검증

#### 4-1. 발밑 타원 그림자
- ✅ 구현 완료
- ✅ 현재 행동하는 캐릭터 명확히 표시
- ✅ 모바일 화면에서 가독성 좋음

#### 4-2. 머리 위 "!" 표시

**이전 문제**: "!"가 깜빡이며 나타났다 사라짐 (UX 불량)
**현재 개선**: 한 개의 "!"가 녹→노→빨로만 색 변경

| 구간 | 색상 | 상태 | 체감 |
|------|------|------|------|
| 0~1.0s | 🟢 녹색 | 가드만 가능 | ✅ 명확 |
| 1.0~2.0s | 🟡 노란색 | 회피·가드 | ✅ 명확 |
| 2.0~2.4s | 🔴 빨간색 | 패링·회피·가드 | ✅ 명확 |

**개선 평가**: ⭐⭐⭐⭐⭐ (5/5)
- 깜빡임 제거로 시선 분산 최소화
- 색상 전환으로 시간 경과 직관적 이해
- 리액션 윈도우 타이밍 정확도 향상

---

### ✅ 5. 패링·회피·가드 버튼

**구현 상태**: ✅ 완료 및 검증

#### 버튼 활성 조건
```
활성 시점: 리액션 윈도우 열렸을 때만
우선순위: PARRY(패링) > DODGE(회피) > GUARD(가드)
연속 사용: 패링 실패 시 패링 제외 → 회피 버튼으로 갱신
```

**실제 플레이 시나리오**:
```
몬스터 공격 → 윈도우 열림 (녹색 구간)
  └─ "가드" 버튼만 활성 (회피/패링 불가)
  
노란색 구간 진입
  └─ "회피" 버튼으로 자동 갱신 (가드/회피 가능)
  
빨간색 구간 진입
  └─ "패링" 버튼으로 갱신 (모두 가능)
  
패링 클릭 → 실패 (타이밍 미스)
  └─ 윈도우 유지 (닫히지 않음)
  └─ 버튼 "회피"로 갱신 (패링 제외)
  └─ 회피 클릭 → 성공!
```

**체감 평가**: ⭐⭐⭐⭐⭐ (5/5)
- 신규 유저도 직관적으로 이해
- 연속 사용으로 재시도 기회 제공
- 실패 → 대안 선택 → 성공의 쾌감

---

### ✅ 6. 턴베이스 손패 유지 (보스전)

**변경 사항**: `player_end_turn()` 시점에 손패를 버리지 않음 → 적 턴 전체 종료 후에만 버림

**이전 문제**: 보스 공격 턴에 손패가 비어 리액션 불가능

**현재 상태**: ✅ 검증 필요 (아직 실제 게임에서 테스트 안 됨)

**예상 UX**:
```
플레이어 턴 (손패: 공격×2, 패링×1, 회피×1)
  ↓
플레이어 턴 종료 (손패 유지)
  ↓
보스 공격 턴 (손패 여전히: 공격×1, 패링×1, 회피×1)
  └─ 리액션 윈도우 열림 → 패링/회피 사용 가능 ✅
  ↓
보스 턴 완료
  ↓
남은 손패 버림 (discard_remaining) ← 여기서만
  ↓
새 플레이어 턴 (새 손패 드로우)
```

**중요**: 이 변경으로 턴베이스 리액션 중요도 ↑

## 🔴 카드 로직 이슈 (조사 중)

### 문제: 카드 사용 시 데미지 미적용

**증상**:
- ✅ 카드 UI 애니메이션: 정상 (사라짐)
- ✅ 손패에서 제거: 정상
- ❌ 몬스터 HP 변화: **없음** (데미지 미적용)
- ❌ 데미지 숫자 표시: 미출력 (결과적으로)

**조사 경로** (Atlas가 작성한 CARD_LOGIC_DEBUG_ANALYSIS.md 참조):
1. `CombatBottomUI._play_card_with_target()` → 신호 emit ✅
2. `InRun_v4._process_next_card_animation()` → 애니 재생 ✅
3. Tween 콜백 → `manager.player_play_card(card, target_index)` 호출 ✅
4. `CombatManagerATB._resolve_card_effect(card, target_index)` **← 데이터 도달 여부 불명**

**의심 지점** (5가지):
1. `card.type != "ATK"` (카드 데이터 오류)
2. `target_index` 범위 벗어남 (인덱스 매핑 오류)
3. `enemies` 배열 비어있음 (몬스터 초기화 실패)
4. 신호 연결 실패 (`hand_updated` 미발생)
5. `target_enemy == null` (타겟 찾기 실패)

**현재 상태**: 🔴 **디버깅 로그 추가 필요**
