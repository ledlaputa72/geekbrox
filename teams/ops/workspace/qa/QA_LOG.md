# QA 로그 (QA_LOG)

> 서브에이전트는 이 파일을 읽고 QA 현황과 버그 이력을 파악한다.

## 상태

## QA 프로세스 (개발 착수 후 적용)
```
기능 테스트 → 밸런스 테스트 → UX 테스트 → 버그 리포트 → 개발팀 수정 → 재검수
```

## 테스트 체크리스트 (Phase 3 알파 빌드부터)

### 기능 테스트
- [ ] 핵심 루프 정상 동작
- [ ] 카드 시스템 버그 없음
- [ ] 방치 보상 계산 정확성
- [ ] 저장/불러오기 정상

### 밸런스 테스트
- [ ] 초반 난이도 (튜토리얼 완료율)
- [ ] 카드 밸런스 (특정 카드 OP 여부)
- [ ] 방치 보상 밸런스 (과다/부족 여부)

### UX 테스트
- [ ] 튜토리얼 이해도
- [ ] UI 직관성
- [ ] 모바일 화면 크기 대응
