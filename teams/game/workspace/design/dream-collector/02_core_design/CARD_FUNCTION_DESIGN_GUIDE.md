## 1. 카드 효과 생성 매트릭스 (v2.0 요약)

모든 카드는 `[비용/조건] → [결과]` 구조를 따르며, '결과'는 `[대상]의 [자원]을(를) [값]만큼 [행동]`하는 하나 이상의 **'카드 효과 원자(Atom)'** 로 구성됩니다.

### 🧩 매트릭스 구성 요소

| 요소 | 종류 (예시) |
| :--- | :--- |
| **① 대상 (Target)** | 나, 단일 적, 모든 적, 무작위 적, 손패, 덱, 버림패, 소멸패... (10+) |
| **② 자원 (Resource)** | HP, 방어도, 에너지, 힘, 민첩, ATB속도, 취약, 카드, 상태이상... (15+) |
| **③ 행동 (Action)** | 얻는다, 잃는다, 준다, 부여한다, 뽑는다, 버린다, 소멸시킨다, 반사한다... (10+) |
| **④ 값 (Value)** | 소(S), 중(M), 대(L), 특대(XL), 변수(X) (5 등급) |

## 2. 생성 가능한 카드 기능 목록 (카테고리별 분류)

이 목록은 매트릭스의 조합으로 생성된 구체적인 카드 기능들을 6가지 핵심 카테고리로 분류한 것입니다.

> **기능 카테고리 ↔ 카드 타입 매핑**

| 기능 카테고리 | 기능 ID 범위 | 주로 사용하는 카드 타입 |
|-------------|-----------|-------------------|
| ⚔️ ① 공격형 (Offensive) | ATK-01~09 | **ATTACK** |
| 🛡️ ② 방어형 — GUARD (Block/Heal) | DEF-01~08 | **SKILL (GUARD 태그)** |
| 🛡️ ② 방어형 — PARRY (패링) | PAR-01~05 | **SKILL (PARRY 태그)** |
| 🛡️ ② 방어형 — DODGE (회피) | DOD-01~05 | **SKILL (DODGE 태그)** |
| 💪 ③ 강화형 (Buff/Scaling) | BUF-01~08 | **POWER** |
| 💀 ④ 약화형 (Debuff/Curse) | DEB-01~07 | **CURSE** |
| 🌀 ⑤ 카드 조작 (Manipulation) | MAN-01~08 | 모든 타입 (부가 효과로 사용) |
| ✨ ⑥ 특수/복합 (Special/Hybrid) | SPC-01~07 | 모든 타입 (복합 효과 카드) |

> **SKILL 타입은 GUARD / PARRY / DODGE 세 계열로 구분됩니다.**
> - GUARD: 즉시 블록·회복 — 리액션 윈도우 오토 가능
> - PARRY: 빨강 구간(0.4초) 반응 → 성공 시 에너지 보상 — 오토 불가 수동 전용
> - DODGE: 노랑+빨강 구간(1.4초) 반응 → 성공 시 에너지 보상 — 오토 성공률 45~60%

| **DEB-06** | CUR_003 | 악마의 계약 | 0 | 에너지 +3 즉시 획득 | 다음 턴 에너지 -2 | — | RARE |
| **DEB-03+07** | CUR_004 | 죽음 | 2 | 적 ATB 50% 감소 + WEAK 3 | — | MAJOR_ARCANA | RARE |
| **DEB-04** | CUR_005 | 탑의 균열 | 1 | 적 블록/방어 완전 무시 (1회) | — | MAJOR_ARCANA | SPECIAL |
| **DEB-05** | CUR_006 | 달의 혼란 | 2 | 적 다음 공격을 자신에게 반사 | 실패 시 플레이어 +20% 피해 | MAJOR_ARCANA | LEGENDARY |

## 5. 카드 생성 규칙 (신규 카드 설계 가이드라인) ⭐ NEW v3.0

> 새로운 카드를 설계할 때 반드시 이 섹션을 참고합니다.
> 기능 ID 조합 → 타입 결정 → 비용 산정 → 밸런스 체크 순서로 진행합니다.

---

### 5.1 카드 생성 플로우

```
1. 컨셉 정의
   └─ "이 카드는 무엇을 하는 카드인가?"
       예: "공격하면서 적을 약화시키는 카드"

2. 기능 ID 선택 (Section 2 참조)
   └─ 예: ATK-01 (단일 공격) + DEB-07 (약화 부여)

3. 타입 결정 (규칙 참조)
   ├─ 주 효과가 공격이면 → ATTACK
   ├─ 주 효과가 방어/회피/패링이면 → SKILL
   ├─ 주 효과가 전투 전체 지속이면 → POWER
   └─ 주 효과가 적 디버프이면 → CURSE

4. 서브 분류 결정 (SKILL만 해당)
   └─ 리액션 윈도우 사용 여부 → GUARD / PARRY / DODGE 태그

5. 비용 산정 (5.2 기준표 참조)

6. 밸런스 체크 (5.3 체크리스트 참조)

7. 카드 ID 할당 및 문서 등록
   └─ CARD_CATALOG_v2.md 업데이트
```

---

### 5.2 타입별 비용 기준표

#### ATTACK 비용 기준

| 비용 | 기대 데미지 | 부가 효과 허용 범위 |
|------|-----------|------------------|
| 1 | 6~8 | 없음 또는 소형 부가 효과 |
| 2 | 9~15 | 드로우 1 또는 약한 디버프 1턴 |
| 3 | 15~18 | 중형 부가 효과 (체력 +5, 드로우 1+α) |
| 4 | 18~22 | 강력한 부가 효과 (블록 +10, 전체 약화 등) |

> **원칙**: 부가 효과가 강할수록 데미지는 낮게 설계 (ATK_003 마법사: 비용2, 데미지4 → 드로우+에너지 보상)

#### SKILL 비용 기준

| 비용 | GUARD (블록) | PARRY (패링 성공 시) | DODGE (회피 성공 시) |
|------|------------|-------------------|------------------|
| 0 | — | 에너지 +1~2 또는 드로우 1 | 에너지 +1 |
| 1 | 블록 5~7 | 에너지 +1 + 반격 | 에너지 +1 + 디버프 |
| 2 | 블록 8~12 | — | — |
| 3 | 블록 15~18 | — | — |

> **원칙**: PARRY/DODGE는 0비용이 기본 (성공 전까지 에너지 소모 없음). GUARD는 비용 = 블록 효율

#### POWER 비용 기준

| 비용 | 지속 효과 강도 | 예시 |
|------|-------------|------|
| 0 | 약한 지속 버프 | 매 턴 드로우 +1 (바보) |
| 1 | 중간 지속 버프 | 공격력 +2 스택, 매 턴 블록 +3 |
| 2 | 중강 지속 버프 | ATB 15% 가속, 에너지 회복 +20% |
| 3 | 강한 지속 버프 | 카드당 에너지 0.5 회복 |
| 4 | 게임 체인저 | 패링 반격 +100% |

> **원칙**: POWER는 전투 전체에 영향을 미치므로 동일 수치 즉시 효과보다 비용이 1~2 높아야 함

#### CURSE 비용 기준

| 비용 | 디버프 강도 | 예시 |
|------|-----------|------|
| 0 | 리스크 동반 디버프 | 취약 (본인도 25% 피해 증가), 악마의 계약 (에너지 선차용) |
| 1 | 3~4턴 약화 또는 1회 강 디버프 | 약화 (공격력 -4, 3턴), 탑의 균열 (블록 무시 1회) |
| 2 | 2~3턴 강 디버프 또는 즉시 판도 변화 | 죽음 (ATB -50% + WEAK 3), 달의 혼란 (공격 반사) |
| 3+ | 전투 판도 변경 수준 | 미래 신규 카드용 |

> **원칙**: 부작용(리스크)이 있는 카드는 비용 0 또는 1이 가능. 순수 디버프는 비용 1~2 필수.

### 5.3 타입별 설계 규칙

#### ATTACK 카드 설계 규칙

```
✅ 반드시 지켜야 할 것:
- 직접 데미지를 주는 효과가 주 효과여야 함
- 타입을 ATTACK으로 설정
- 리액션 윈도우에서 사용 불가 (태그 없음)
- 사용 후 버림 더미로 이동

✅ 허용되는 부가 효과:
- 드로우 1~2장
- 자기 자신에게 버프 (체력 회복, 블록)
- 적에게 단순 디버프 1개 (WEAK, VULNERABLE)
- MAJOR_ARCANA 태그 추가 가능

❌ ATTACK에 넣으면 안 되는 효과:
- 전투 전체 지속 버프 → POWER로 이동
- 적 ATB 조작, 공격 반사 등 복잡한 디버프 → CURSE로 이동
- 패링/회피 판정 → SKILL로 이동
```

#### SKILL 카드 설계 규칙

```
✅ 반드시 지켜야 할 것:
- 방어(블록), 패링, 회피 중 하나가 주 효과
- 타입을 SKILL로 설정
- 태그 GUARD / PARRY / DODGE 중 반드시 하나 이상 부여
- PARRY 카드: 오토 모드에서 비활성(✕) 처리 필수
- DODGE 카드: 오토 성공률(45~60%) 명시 필수
- 사용 후 버림 더미로 이동

✅ 서브 분류별 특징:
- GUARD: 즉시 블록 제공, 오토 발동 가능, 비용 1~3
- PARRY: 성공 보상 중심, 0비용 기본, 높은 난이도 가정
- DODGE: 성공 보상 + 오토 성공률, 0비용 기본

❌ SKILL에 넣으면 안 되는 효과:
- 전투 전체 지속 버프 → POWER로 이동
- 직접 공격 데미지가 주 효과 → ATTACK으로 이동
```

#### POWER 카드 설계 규칙

```
✅ 반드시 지켜야 할 것:
- 효과가 전투 전체에 지속되어야 함 (즉시 소모 효과 아님)
- 타입을 POWER로 설정
- 사용 시 active_powers 목록에 등록 (버림 더미로 가지 않음)
- 중첩 가능 여부 명시 (스택 가능/불가)
- 트리거 조건 명시 (매 턴 시작 / 카드 플레이 시 / 패링 성공 시 등)

✅ 허용되는 지속 효과 유형:
- 매 턴 자원 회복 (블록/에너지/드로우 자동 생성)
- 기본 스탯 증가 (STRENGTH, 민첩 등)
- 특정 행동 시 추가 보상 (패링 성공 시, 카드 사용 시)
- 게임 속도 변경 (ATB 가속/감속)

❌ POWER에 넣으면 안 되는 효과:
- 즉시 피해 (ATTACK 기능)
- 즉시 블록 생성이 전부인 카드 (SKILL로 이동)
- 적에게 직접 디버프 (CURSE로 이동)
```

#### CURSE 카드 설계 규칙

```
✅ 반드시 지켜야 할 것:
- 적에게 불리한 효과가 주 효과여야 함
- 타입을 CURSE로 설정
- 타로 "어두운 아르카나" 테마 반영 권장 (악마/죽음/탑/달 등)
- 리스크/리워드 구조 카드: 부작용을 카드 설명에 명확히 표시

✅ 허용되는 효과 유형:
- 적 공격력 감소 (약화 계열)
- 적이 받는 피해 증가 (취약 계열)
- 적 ATB/속도 조작
- 적 블록 무시/관통
- 적 공격 반사
- 플레이어 에너지 선차용 (에너지 선획득 + 다음 턴 패널티)

✅ 리스크/리워드 설계 기준:
- 리스크 없는 카드: 비용 1~2
- 리스크 있는 카드 (플레이어 대가 필요): 비용 0~1 가능
- 대형 리스크 카드: 비용 0 + LEGENDARY 희귀도

❌ CURSE에 넣으면 안 되는 효과:
- 직접 데미지 (ATTACK으로 이동)
- 자기 자신에게 버프 (POWER/ATTACK의 부가 효과로 처리)
- 전투 전체 지속 자기 버프 (POWER로 이동)
```

### 5.4 밸런스 체크리스트

새로운 카드를 설계한 후 아래 체크리스트를 확인합니다:

```
□ 비용 대비 기대값이 동일 타입 동일 비용 카드 범위 내인가?
□ 너무 강한 부가 효과가 메인 효과와 결합되어 있지 않은가?
□ 희귀도가 효과 강도와 일치하는가? (COMMON < RARE < SPECIAL < LEGENDARY)
□ MAJOR_ARCANA 태그 적용 여부를 결정했는가?
  └─ 기준: 타로 주요 카드 테마 + 강력한 효과 조합
□ ATB / 턴베이스 양쪽 모드에서 효과가 합리적인가?
□ POWER 카드의 경우: active_powers 중첩 시 수치가 너무 높아지지 않는가?
□ CURSE 카드의 경우: 리스크/리워드 비율이 적절한가?
□ SKILL PARRY 카드의 경우: 오토 비활성(✕) 설정이 되어 있는가?
□ SKILL DODGE 카드의 경우: 오토 성공률(45~60%)이 설정되어 있는가?
□ 덱 구성 권장 비율 내에서 이 카드의 역할이 명확한가?
```

---

### 5.5 런타임 덱 구성 권장 비율

> ⚠️ **중요 용어 구분**:
> - **카드 풀**: 42장 (v2.0 — 게임에 존재하는 전체 카드)
> - **런타임 덱**: 플레이어가 로그라이크 런 중 직접 구성하는 덱 (일반적으로 20~35장)
> - 아래 표는 **런타임 덱 30장 기준** 권장 비율입니다 (카드 풀 42장과 다름)

| 빌드 스타일 | ATTACK | SKILL(GUARD) | SKILL(PARRY) | SKILL(DODGE) | POWER | CURSE | 합계 |
|-----------|--------|-------------|-------------|-------------|-------|-------|------|
| 공격형 | 15 | 5 | 3 | 2 | 3 | 2 | 30 |
| 방어형 | 8 | 10 | 4 | 1 | 5 | 2 | 30 |
| 패링 특화형 | 10 | 3 | 8 | 0 | 4 | 5 | 30 |
| 회피 특화형 | 10 | 3 | 0 | 8 | 4 | 5 | 30 |
| 파워/버프형 | 10 | 5 | 2 | 1 | 10 | 2 | 30 |
| 저주/약화형 | 10 | 5 | 3 | 2 | 4 | 6 | 30 |

> **신규 카드 추가 시**: 위 비율에서 어느 빌드 스타일을 강화하는 카드인지 먼저 판단 후 설계.
> **SKILL 3계열 비율 기준**: 방어형 덱은 GUARD 중심, 패링 특화는 PARRY 집중, 혼합형은 균등 분배.

### 5.6 카드 ID 명명 규칙

| 타입 | ID 형식 | 범위 | 예시 |
|------|--------|------|------|
| ATTACK | ATK_XXX | 001~ | ATK_011 (M2 첫 번째 공격) |
| SKILL (GUARD) | DEF_XXX | 001~ | DEF_009 (M2 첫 번째 가드) |
| SKILL (PARRY) | PAR_XXX | 001~ | PAR_006 (M3 패링 추가) |
| SKILL (DODGE) | DOD_XXX | 001~ | DOD_006 (M4 회피 추가) |
| POWER | POW_XXX / SKL_XXX | 001~ | POW_007 (M2 파워) |
| CURSE | CUR_XXX | 001~ | CUR_007 (M2 커스) |

> **SKL_XXX**: 기존 카드와의 호환성을 위해 바보(SKL_001), 달의 환영(SKL_002) 유지. 신규 POWER 카드는 POW_XXX 사용.
