# 🃏 Dream Collector - 카드 기능 설계 가이드 v2.2

**버전**: v2.2 (2026-03 업데이트 — 4타입 체계 v2.0 정렬)
**문서 목적**: '카드 기능 생성 매트릭스'를 기반으로, 게임에 구현될 카드의 모든 기능을 유형별로 분류하고 구체적인 예시를 제시하여 개발팀이 참조할 수 있는 기획 문서를 제공합니다.

> **v2.2 업데이트 내용**:
> - 카드 타입 명칭을 v2.0 기준으로 정렬 (ATK→ATTACK, DEF/PARRY/DODGE→SKILL, SKILL→POWER, 신규 CURSE)
> - "원형" 시스템(꿈의 분노/고요한 방벽/뒤틀린 악몽/샘솟는 영감)은 캐릭터 빌드 참고용으로 유지
> - CURSE 카드 기능 카테고리 명확화 (④ 약화 기능이 CURSE 타입의 주 효과임)
> - 현재 42장 카드와 매핑 추가 (각 기능 ID별 현재 카드 예시)

## 1. 카드 효과 생성 매트릭스 (v2.0 요약)

모든 카드는 `[비용/조건] → [결과]` 구조를 따르며, '결과'는 `[대상]의 [자원]을(를) [값]만큼 [행동]`하는 하나 이상의 **'카드 효과 원자(Atom)'** 로 구성됩니다.

### 🧩 매트릭스 구성 요소

| 요소 | 종류 (예시) |
| :--- | :--- |
| **① 대상 (Target)** | 나, 단일 적, 모든 적, 무작위 적, 손패, 덱, 버림패, 소멸패... (10+) |
| **② 자원 (Resource)** | HP, 방어도, 에너지, 힘, 민첩, 악몽, 취약, 카드, 상태이상... (15+) |
| **③ 행동 (Action)** | 얻는다, 잃는다, 준다, 부여한다, 뽑는다, 버린다, 소멸시킨다... (10+) |
| **④ 값 (Value)** | 소(S), 중(M), 대(L), 특대(XL), 변수(X) (5 등급) |

## 2. 생성 가능한 카드 기능 목록 (카테고리별 분류)

## 3. 카드 타입별 기능 매핑 (v2.0 기준)

4가지 카드 타입과 위 기능 카테고리의 관계:

| 카드 타입 | 코드 | 주로 사용하는 기능 카테고리 | 현재 카드 예시 |
|---------|------|----------------------|------------|
| **ATTACK** | `"ATTACK"` | ① 공격형 (ATK-01~09) | ATK_001~010 |
| **SKILL** | `"SKILL"` | ② 방어형 (DEF-01~08) + 태그 기반 리액션 | DEF_001~008, PAR_001~005, DOD_001~005 |
| **POWER** | `"POWER"` | ③ 강화형 (BUF-01~08) — 전투 전체 지속 | SKL_001, SKL_002, POW_001~006 |
| **CURSE** | `"CURSE"` | ④ 약화형 (DEB-01~07) — 적에게 적용 | CUR_001~006 |

> **주의**: CURSE 타입은 v1의 "플레이어에게 해악을 끼치는 저주" 개념과 다릅니다.
> v2.0에서는 **적에게 불리한 디버프/저주를 거는 카드**. 일부는 리스크/리워드 구조.
> 플레이어에게 악영향을 주는 것이 아닌, 적에게 저주를 거는 공격 수단입니다.

### 기능별 현재 카드 연결

**① 공격형** — ATTACK 카드
- ATK-01 (단타): ATK_001 검의 에이스, ATK_006 번개
- ATK-02 (광역): ATK_003 마법사, ATK_007 악마
- ATK-03 (다중): ATK_002 이중 베기
- ATK-04 (자해 고수익): ATK_004 탑, ATK_005 세계
- ATK-09 (공격+디버프): ATK_007 악마 (독 부여)

**② 방어형** — SKILL 카드 (GUARD 태그)
- DEF-01 (기본 블록): DEF_001~007
- DEF-08 (회복): DEF_008 절제

**③ 강화형** — POWER 카드
- BUF-01 (영구 힘): POW_001 힘, ATK_010 황제 (ATTACK이지만 STRENGTH 부여)
- BUF-05 (에너지): POW_005 연인의 유대
- BUF-06 (드로우 강화): SKL_001 바보 (매 턴 드로우+1)

**④ 약화형** — CURSE 카드 (적에게 적용)
- DEB-01 (단일 취약): CUR_002 취약 (적 피해 +50%)
- DEB-02 (광역 약화): CUR_001 약화 (적 공격력 -4)
- DEB-05 (공격력 감소): CUR_001 약화

## 4. 개발 반영을 위한 다음 단계

### 1. **JSON 스키마 설계**
위 기능 ID와 매트릭스를 기반으로 `cards.json`에 들어갈 구체적인 스키마를 설계합니다.

**예시 `effects` 필드 구조**:
```json
"effects": [
  {
    "action": "DEAL",
    "target": "E1",
    "resource": "HP",
    "value": "M",
    "value_concrete": 6
  },
  {
    "action": "APLY",
    "target": "E1",
    "resource": "VUL",
    "value": "M",
    "value_concrete": 1
  }
]
```

### 2. **CardEffectSystem 구현**
`CardEffectSystem.gd`에서 위 기능 ID 또는 `action`, `target`, `resource` 조합에 따른 실제 게임 로직을 구현하는 `switch` 또는 `match` 문을 작성합니다.

### 3. **카드 리스트 작성 (스프레드시트)**
개발과 동시에, 기획자는 위 기능 목록을 참고하여 4대 원형과 등급에 맞춰 실제 카드 이름과 스토리를 부여하는 작업을 진행합니다.

**스프레드시트 컬럼**:
`카드 이름`, `원형`, `기능 ID (조합)`, `비용`, `효과 설명`, `레퍼런스/스토리`
