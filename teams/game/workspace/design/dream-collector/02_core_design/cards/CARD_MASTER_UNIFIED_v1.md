## 📊 **카드 시스템 구조**

### 1️⃣ **공격력 (ATTACK) — 10장** ✅ 완성
• ATK-SGL: 단일 타격 (4장)
• ATK-MLT: 광역·다중 (3장)
• ATK-RCK: 고위험 (1장)
• ATK-CMP: 복합 효과 (2장)

### 2️⃣ **방어 (SKILL) — 18장** ✅ 완성
• SKL-GRD: GUARD 즉시 블록 (8장)
• SKL-PAR: PARRY 패링 윈도우 (5장)
• SKL-DOD: DODGE 회피 윈도우 (5장)

### 3️⃣ **강화 (POWER) — 8장** (2장 ✅ + 6장 🔲)
• PWR-DRW: 드로우 (2장) ✅
• PWR-ATK: 공격력 강화 (2장) 🔲
• PWR-DEF: 방어 강화 (2장) 🔲
• PWR-SUS: 지속 효과 (2장) 🔲

### 4️⃣ **약화 (CURSE) — 6장** 🔲 (설계 대기)
• CRS-STA: 스탓 약화 (2장)
• CRS-SPD: 속도 조작 (1장)
• CRS-PEN: 관통/반사 (2장)
• CRS-RSK: 리스크/기타 (1장)

**합계**: 30장 ✅ (구현) + 12장 🔲 (설계) = **42장**

## 🔑 **ID 체계 규칙**

### **기능 ID 명명 규칙**
```
메인-서브 형태

ATTACK (ATK)
  ├─ ATK-SGL (Single): 단일 타격
  ├─ ATK-MLT (Multi): 광역·다중
  ├─ ATK-RCK (Risk): 고위험
  └─ ATK-CMP (Combo): 복합 효과

SKILL (SKL)
  ├─ SKL-GRD (Guard): 즉시 블록
  ├─ SKL-PAR (Parry): 패링 윈도우
  └─ SKL-DOD (Dodge): 회피 윈도우

POWER (PWR)
  ├─ PWR-DRW (Draw): 드로우
  ├─ PWR-ATK (Attack): 공격력 강화
  ├─ PWR-DEF (Defense): 방어 강화
  └─ PWR-SUS (Sustain): 지속 효과

CURSE (CRS)
  ├─ CRS-STA (Stat): 스탯 약화
  ├─ CRS-SPD (Speed): 속도 조작
  ├─ CRS-PEN (Penetrate): 관통/반사
  └─ CRS-RSK (Risk): 리스크/기타
```

### **카드 ID 명명 규칙**
```
타입_번호 형태

ATTACK: ATK_001 ~ ATK_010 (10장)
SKILL: DEF_001~008, PAR_001~005, DOD_001~005 (18장)
POWER: SKL_001~002, POW_001~006 (8장)
CURSE: CUR_001~006 (6장)
```

### **코드 생성 시 사용 예시**
```python
# 기능 ID로 카드 검색
cards = db.get_by_function_id("ATK-SGL")

# 서브 카테고리로 카드 검색
```
