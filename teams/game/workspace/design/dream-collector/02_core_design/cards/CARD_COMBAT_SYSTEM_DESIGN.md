## 📋 문서 개요

이 문서는 Dream Collector의 **카드 기반 전투 시스템**의 완전한 설계를 정의합니다.

- **핵심 콘셉트**: 플레이어의 반응성과 전략성을 모두 만족하는 이중 전투 모드
- **대상 게이머**: 액션 게이머(ATB) + 전략 게이머(턴베이스)
- **구현 상태**: 기획 확정, 개발 진행 중(88% 완료)

## 🎯 디자인 철학

### 1. 에너지 = 자원 관리
```
핵심: 에너지는 "더 공격할 수 있는 기회"

방어 우수 플레이어:
  방어 성공 → 에너지 보너스 → 더 공격 가능
  = 반사신경 높은 플레이어가 실제로 강함

결과: 방어를 잘할수록 더 강해지는 긍정적 피드백
```

### 2. 이중 시스템
```
ATB (실시간): 액션 게이머 만족
  - 빠른 반응
  - 스트레스 높음
  - 보상: 속도감 있는 게임플레이

턴베이스 (전략): 전략 게이머 만족
  - 충분한 사고 시간
  - 스트레스 낮음
  - 보상: 깊이 있는 전략성
```

### 3. 접근성
```
오토 플레이(FULL AUTO):
  - 영상 미감상용 자동 플레이
  - 강제성 없음 (게임을 언제든 멈추고 직접 조작 가능)

세미 오토(SEMI):
  - AI가 추천만 함
  - 플레이어가 최종 결정

수동(MANUAL):
  - 풀 컨트롤
```

**희귀도**:
- `COMMON`: 기본 카드
- `RARE`: 희귀 카드

## ⚡ 에너지 시스템

### ATB 에너지 (실시간)

**기본값**: 3 (최대 5, 오버플로우)

**자동 충전**:
- 5초마다 +1 (시간 기반)
- 모든 엔티티(플레이어/적) 동일

**방어 보상** (즉시):
```
패링 성공: +2 (가장 좋음)
회피 성공: +1
방어 성공: +0.5
```

**오버플로우**:
- 3초간 유지 (그 후 3으로 강제 복구)
- 시각: 황금 테두리

---

### 턴베이스 에너지 (전략적)

**기본값**: 턴당 3 (최대 5, 오버플로우)

**지급**: 플레이어 턴 시작 시 고정 지급

**방어 보상** (다음 턴에 적용):
```
패링 성공: 다음 턴 +2 추가
회피 성공: 다음 턴 +1 추가
방어 성공: 보너스 없음 (블록만)
```

**예시**:
```
턴 1: 에너지 3
     카드 3장 사용 (모두 소비)
     적 공격 → 회피 성공!

턴 2: 에너지 3 + 1(회피보너스) = 4
     카드 4장 사용 가능
```

## 🎯 리액션 시스템 (공통)

### 4가지 반응 타입

```
1. PARRY (패링)
   조건: 공격 무효화 가능 카드 사용 + 0.8초 내 탭
   결과:
   - 피해: 0
   - ATB: 에너지 +2 즉시, 적 ATB 50% 롤백
   - 턴베이스: 다음 턴 에너지 +2 추가

2. DODGE (회피)
   조건: 회피 카드 사용 + 1.8초 내 탭
   결과:
   - 피해: 0
   - ATB: 에너지 +1 즉시
   - 턴베이스: 다음 턴 에너지 +1 추가

3. GUARD (방어)
   조건: 방어 카드 사용
   결과:
   - 피해: 블록으로 상쇄
   - ATB: 에너지 +0.5 즉시
   - 턴베이스: 에너지 보너스 없음

4. NONE (무반응)
   조건: 카드 미사용 또는 시간 초과
   결과:
   - 피해: 풀 데미지
   - 보너스: 없음
```

### 특수: UNBLOCKABLE (관통 공격) 🔱

```
특징: 패링 불가능! 회피만 가능!

시각:
- 🔱 아이콘 표시
- "방어 불가! 회피하세요!" 텍스트 (빨간색)
- 패링 카드 회색 처리

의도:
- 방어 전략을 깨부수는 공격
- 플레이어에게 새로운 도전 제시
```

### 리액션 윈도우

```
패링 윈도우:
  Story 모드 (기본): 0.8초
  Hard 모드: 0.5초

회피 윈도우:
  Story 모드 (기본): 1.8초
  Hard 모드: 1.2초

규칙: 패링 → 회피 순서
      즉, 0.8초 안에 패링하면 "패링"
      그 후 1.8초까지 회피 가능
```

## 🎪 콤보 시스템

### 4가지 콤보

```
1. 연타 (Triple Hit)
   조건: ATK 카드 3장 연속
   보너스: +75% 데미지
   예: 6 + 6 + 6 = 18 → 31.5 데미지
   
2. 패링 반격 (Parry Counter)
   조건: PARRY 카드 → ATK 카드 직후
   보너스: +30% 데미지
   예: 패링 후 8 데미지 → 10.4 데미지
   의도: 방어 후 즉시 반격하는 플로우

3. 약점 폭로 (Exploit Weakness)
   조건: 적이 VULNERABLE 상태 + ATK 카드
   보너스: +50% 데미지
   예: 10 데미지 → 15 데미지
   의도: 약화된 적을 집중 공격

4. 완벽한 방어 (Perfect Guard)
   조건: DEF 카드 2장 연속
   보너스: 없음 (블록 누적만)
   예: 7 + 12 = 19 블록
   의도: 방어를 연속으로 해도 보상은 없음
```

### 콤보 시스템 철학

```
목적: 플레이어가 카드를 "계획적으로" 배열하게 함

효과:
1. 사전 계획 중요성: 다음 행동 예측
2. 덱 빌드 영향: 어떤 카드를 포함할지 중요
3. 피드백: 좋은 조합에 대한 보상 제시
```

## 😊 상태이상 시스템 (7가지)

```
1. POISON (중독)
   - 매 턴 HP 감소
   - 스택 가능 (중독3 = -3 HP/턴)

2. VULNERABLE (취약)
   - 받는 데미지 +50%
   - 보스전 핵심 전략

3. WEAK (약화)
   - 주는 데미지 -25%
   - 양쪽 모두 약화 가능

4. STRENGTH (힘)
   - 공격력 증가
   - 버프 시스템

5. DEXTERITY (민첩)
   - 블록 획득량 증가
   - 방어 강화

6. BURNING (화상)
   - 중독처럼 매턴 데미지
   - 시각적으로 구분

7. ENTANGLED (묶음)
   - 이번 턴 공격 불가
   - 무력화 상태
```

## 🌊 ATB 전투 상세 설계

### ATB 게이지 (실시간)

```
범위: 0 ~ 100
충전: speed 기반 (speed 100 = 1초에 1 충전)

플레이어 speed: 70~100 (조정 가능)
적 speed: 다양함

규칙:
- ATB 100 도달 → 행동 실행 → ATB 초기화
- 한 프레임에 한 엔티티만 처리 (동시 행동 방지)
```

### ATB 의도 표시

```
ATB 0~60%: 일반 표시 (아이콘 작음)
ATB 60~80%: 강조 표시 (아이콘 커짐)
ATB 80~95%: 경고 표시 (주황 테두리)
ATB 95%+: 위험 표시 (빨간색 + 진동 + "⚡위험!")

의도: 플레이어가 적의 공격 임박을 직관적으로 파악
```

### ATB 특수 모드

**집중 모드 (Focus Mode)**
```
조건: 플레이어가 에너지 1 소모하여 활성화

효과:
- 시간 흐름: 0.3배 슬로우
- 모든 적의 공격이 3배 느려짐 (상대적으로)
- 리액션 타이밍이 3배 쉬워짐

지속: 3초 (게임 시간)
비용: 에너지 1 (한 번 사용하면 자동 종료)

의도: 어려운 공격에 대한 플레이어 능력 제공
      게임 접근성 향상
```

**위기 모드 (Crisis Mode)**
```
조건: 플레이어 HP 30% 이하 (자동 발동)

효과:
- 시간 흐름: 0.5배 슬로우
- 적의 공격이 2배 느려짐
- 플레이어가 회복할 기회 제공

지속: 10초 (게임 시간)
비용: 자동 (무료)

시각:
- 화면 가장자리 빨간 테두리
- 맥박 효과 (주기적 깜빡임)
- "위기!" 텍스트

의도: 플레이어가 죽음 직전에 역전 가능하게 함
      게임을 더 관대하게 (괜찮은 흐름)
```

## 📋 턴베이스 전투 상세 설계

### 턴 루프

```
[플레이어 턴]
- 에너지 3 또는 보너스 포함
- 카드 여러 장 사용 가능 (에너지 범위 내)
- 패스하면 턴 종료

[적 턴들]
- 모든 적이 의도된 행동 실행
- 플레이어는 리액션 윈도우에서만 행동 가능
- 모든 적 행동 완료 후 턴 종료

[상태이상 처리]
- POISON 등 지속 효과 처리
- 카운트 감소

[다음 플레이어 턴]
- 카드 드로우
- 상태 초기화
```

### 손패 시스템

```
드로우: 턴당 5장
최대: 10장
초과: 드로우 불가

버림더미:
- 사용한 카드가 들어감
- 모든 카드 사용 후 덱이 비면 자동 셔플

규칙:
- 턴 시작 시 드로우
- 턴 종료 시 미사용 카드 모두 버림
```

### 턴베이스 의도 표시

```
현재 공격: 크게 표시
├─ 아이콘 (공격 타입)
├─ 데미지 값
└─ 상태이상 효과

다음 1행동: 중간 크기
다음 2행동: 작은 크기

아이콘 매핑:
⚔️: 일반 공격
⚔️⚠️: 강한 공격
🌀: 광역 공격
🔱: 관통 공격 (패링 불가)
✨: 버프
🛡️: 방어

의도: 플레이어가 미리 대비할 수 있게 함
      턴제 게임의 전략성 강조
```

### 턴베이스 보조 시스템

**타로 에너지 (Tarot Energy)**
```
최대: 3
충전: 메이저 아르카나 카드 사용 시 +1

용도:
- 강력한 카드 비용 지불
- 예: "달의 환영" = 타로 2 소모

의도: 강력한 카드를 더 전략적으로 사용하게 함
```

**꿈 조각 (Dream Shard)**
```
최대: 5

획득:
- 패링 성공 시 +1
- 특정 콤보 완성 시 +1

사용 능력 (4가지):
1. QUICK_DRAW (1 소모): 카드 1장 추가 드로우
2. ENERGY_BURST (2 소모): 에너지 +1
3. DREAM_HEAL (3 소모): HP +8 치유
4. NIGHTMARE (5 소모): 모든 적에 VULNERABLE +2

의도: 방어를 잘할수록 강력해짐
      위기 상황에서 역전 가능
```

**덱 패시브 (Deck Passive)**
```
조건: 덱 구성에 따라 자동 적용

예시:
- DEF 5장+: "달의 기사" → 턴 시작 블록+3
- ATK 7장+: "검의 달인" → 첫 공격 데미지+2
- 메이저 아르카나 3장+: "타로 학자" → 전투 시작 타로+1
- PARRY 4장+: "달빛 반격사" → 패링 보너스+1

의도: 덱 빌드에 따라 플레이 스타일 다양화
      같은 덱도 다양한 전략 가능
```

## 📊 비교: ATB vs 턴베이스

### 게임 플로우

| 항목 | ATB | 턴베이스 |
|------|-----|---------|
| 시간 개념 | ⏱️ 계속 흐름 | 📋 턴 단위 |
| 플레이어 컨트롤 | 실시간 | 턴 대기 시간 있음 |
| 적 행동 | 동시 진행 | 순차 실행 |
| 사고 시간 | 짧음 | 충분함 |

### 에너지 관리

| 항목 | ATB | 턴베이스 |
|------|-----|---------|
| 충전 방식 | 시간 기반 자동 | 턴 시작 고정 지급 |
| 자동 충전 | 5초마다 +1 | 없음 |
| 방어 보상 | 즉시 적용 | 다음 턴 적용 |

### 게이머 선호도

| 게이머 | 추천 모드 | 이유 |
|--------|---------|------|
| 액션 게이머 | ATB | 빠른 반응, 실시간 피드백 |
| 전략 게이머 | 턴베이스 | 깊이 있는 전략, 사고 시간 |
| 캐주얼 | 둘 다 가능 | 선택 자유 |

## 🎯 핵심 설계 원칙

### 1. 방어 = 약함이 아님
```
패링/회피 성공 → 에너지 보너스 → 더 강한 공격
= 수동적 방어가 아니라 에너지 획득 전략
```

### 2. 이중 시스템, 하나의 게임
```
30장의 카드는 같음
하지만 ATB/TB에서 완전히 다른 게임 경험
= 게이머 성향에 따른 선택
```

### 3. 의도된 조합 (콤보)
```
랜덤이 아니라 플레이어 선택
= 덱 빌드 + 턴 플레이 모두 중요
```

### 4. 접근성
```
모든 게이머 환영
- 액션: ATB로 도전
- 전략: TB로 몰입
- 캐주얼: AUTO로 감상
```
