# 🔧 Dream Collector — 4가지 시스템 핵심 메커니즘 상세 가이드

**작성:** Atlas (PM) → Game팀 (구현 기준)  
**기준:** 특성/세트/강화/성장 시스템의 핵심 + 상호연결 구조  
**버전:** v1.0  
**상태:** 📋 심층 분석 완료

---

## 📌 개요

**목표:** 4가지 시스템이 실제 게임플레이에서 어떻게 작동하고, 서로 어떻게 연결되는지 상세 설명

**구조:**
1. 각 시스템의 핵심 메커니즘
2. 실제 게임플레이 흐름 추적
3. 시스템 간 상호작용 (System Interconnection)
4. 데이터 흐름도 (Data Flow)

## 1️⃣ 특성 시스템 (Trait System)

### 핵심 메커니즘

**5가지 특성의 작동 원리:**

```
특성 = "자동으로 계산되는 보너스"

작동 조건:
  특성은 조건 만족 시 자동으로 활성화
  플레이어가 "특성을 활성화한다"는 개념 없음
  
예시:
  무기를 장착하면 → 무기 마스터리 자동 활성화
  방어구를 장착하면 → 방어 강화 자동 활성화
  
특성 레벨:
  특성 레벨 = 장비 강화도 또는 개수
  
  무기 마스터리 레벨 = 무기 강화도 (+0~+10)
  방어 강화 레벨 = 방어구 강화도
  악세서리 동조 레벨 = 악세서리 개수 (1~3)
  
계산 방식:
  보너스 = 기본값 × (1 + 특성 효과%)
```

### 실제 게임플레이 흐름

```
[전투 시작]
  ↓
[플레이어가 카드 선택]
  ↓
[기본 공격 계산 시작]
  
  기본 데미지 = 무기 ATK (예: 20)
  
  ① 무기 마스터리 적용
     현재: 무기 +5 강화
     마스터리 레벨 5 = +25% 보너스
     데미지 = 20 × 1.25 = 25
  
  ② 악세서리 동조 적용
     현재: 악세서리 2개 착용
     동조 레벨 2 = +10% 시너지
     데미지 = 25 × 1.10 = 27.5
  
  ③ 장비 연쇄 적용
     현재: 검 세트 3/4 완성
     연쇄 보너스 = +15% (3개 조건)
     데미지 = 27.5 × 1.15 = 31.625
  
  ④ 듀얼 웰드 적용 (보조 무기 있으면)
     보조 무기 단검 +3 강화
     추가 데미지 = 8 × 1.75 = 14
     최종 데미지 = 31.625 + 14 = 45.625

[최종 데미지: 약 46 데미지] ✓
```

### 다른 시스템과의 상호작용

```
특성 ← → 장비 시스템
  연결: 특성 레벨 = 장비 강화도/개수
  의존: 특성 효과는 장비에 의존
  영향: 장비 변경 → 특성 효과 즉시 변경

특성 ← → 강화 시스템
  연결: 장비 강화 → 특성 레벨 증가
  의존: 특성 효과 = 장비 강화도 함수
  영향: 강화 1단계 = 특성 효과 5% 증가

특성 ← → 기본 공격 시스템 (ATB)
  연결: 특성 보너스 → 기본 공격 데미지
  의존: 기본 공격 = 특성의 주요 적용 대상
  영향: 특성이 없으면 기본 공격 약함

특성 ← → 성장 시스템
  연결: 마스터리는 자동 성장
  의존: 플레이어 선택 필요 없음
  영향: 성장 = 자동으로 더 강한 특성
```

## 2️⃣ 세트 효과 시스템 (Set Effect System)

### 핵심 메커니즘

**세트의 작동 원리:**

```
세트 = "같은 시리즈 장비를 모으면 추가 보너스"

세트 정의:
  [검 세트] = 검 + 갑옷 + 반지 + 방패
  [마법 세트] = 지팡이 + 로브 + 신발 + 주술서
  ... 등등
  
세트 진행도:
  2개 완성 → 기본 보너스
  3개 완성 → 추가 보너스
  4개 완성 → 최고 보너스
  
보너스는 누적:
  2개: ATK +10%
  3개: ATK +15%, DEF +10% (3개 보너스는 2개의 것들을 포함)
  4개: ATK +20%, DEF +15%, 특수 효과

특성화:
  플레이어는 세트 선택으로 게임 방향 결정
```

### 실제 게임플레이 흐름

```
[장비 획득]
  ↓
[가방 → 장비 장착 화면]

화면 구성:
  
  [세트 진행도]
  
  검 세트: ▓▓▒░
    2/4 완성
    현재: 검(O), 갑옷(O), 반지(X), 방패(X)
    활성 보너스: ATK +10%, 전투력 +5%
    3개 시: DEF +10% 추가 예정
    4개 시: 기본 공격 2회 연속 예정
    [권장: 반지 획득]
  
  마법 세트: ▒░░░
    1/4 완성
    현재: 지팡이(O), 로브(X), 신발(X), 주술서(X)
    활성 보너스: 없음 (2개 필요)
    [권장: 로브 또는 신발 획득]

[플레이어: 반지 장착]
  ↓
[세트 진행도: 3/4]
  ↓
[새로운 보너스 자동 활성화]
  DEF +10% 추가 적용
  ↓
[기본 공격 재계산]
  이전: 46 데미지
  세트 보너스 +15% (2→3개):
  새로운: 46 × 1.15 = 52.9 데미지 ✓
```

### 세트와 다른 시스템의 상호작용

```
세트 ← → 특성 시스템
  연결: 세트 선택 = 플레이 스타일 결정
  의존: 세트와 마스터리가 함께 보너스 제공
  영향: 
    검 세트 + 무기 마스터리 = "검 전사" 빌드
    마법 세트 + 방어 강화 = "마법사" 빌드
  
  예시 계산:
    기본 데미지: 35
    무기 마스터리 (+30%): 35 × 1.30 = 45.5
    검 세트 4/4 (+20%): 45.5 × 1.20 = 54.6 ✓

세트 ← → 강화 시스템
  연결: 각 세트 장비를 강화
  의존: 세트는 강화된 장비로만 효과 극대화
  영향: 같은 세트 4개를 모두 +10까지 강화하려면
         681,000 골드 필요 (전략적 선택)

세트 ← → 성장 시스템
  연결: 플레이어가 세트 목표 설정 → 성장 방향 정해짐
  의존: "다음 세트 장비를 얻기 위해 던전 가기"
  영향: 성장 = 세트 완성을 향한 진행
```

## 3️⃣ 강화 시스템 (Enhancement System)

### 핵심 메커니즘

**강화의 작동 원리:**

```
강화 = "같은 장비를 더 강하게 만들기"

강화도: +0 ~ +10

비용 공식:
  비용 = 1,500 G × 1.4^강화도
  
  +0: 1,500 G
  +1: 2,100 G
  +2: 2,940 G
  ...
  +10: 102,000 G
  
  4개 슬롯 풀강화: 약 400,000 G

효과:
  각 +1 = 스탯 +5~10% 증가

성공률:
  +0~+6: 100% (항상 성공)
  +7: 85% (15% 실패 확률)
  +8: 70% (30% 실패 확률)
  ...
  +10: 40% (60% 실패 확률)
  
  실패 시: 강화도 -1 (한 단계 하락)
```

### 실제 게임플레이 흐름

```
[플레이어가 강화 화면 진입]

현재 무기: 빛의 검 (+7)
통계:
  ATK: 30 × (1 + 0.07×5) = 40.5

강화 버튼 클릭:
  
  강화 요구:
    비용: 2,100 × 1.4^8 = 약 25,600 G (보유: 50,000 G)
    성공률: 70%
  
  선택지:
    [강화하기] [보호권 사용] [취소]
  
  [강화하기] 선택 → 70% 확률
    
    성공한 경우 (70%):
      +7 → +8
      새로운 ATK: 30 × (1 + 0.08×5) = 42
      기본 공격 데미지: 이전 52.9 → 54.6 (+1.7)
      ✓ 성공!
    
    실패한 경우 (30%):
      +7 → +6으로 하락
      새로운 ATK: 30 × (1 + 0.06×5) = 39
      기본 공격 데미지: 이전 52.9 → 51.2 (-1.7)
      비용: 25,600 G 손실
      ✗ 실패! (하지만 다음번엔 +7로 재도전 가능)
```

### 강화와 다른 시스템의 상호작용

```
강화 ← → 특성 시스템
  연결: 장비 강화 = 특성 레벨 증가
  의존: 특성 효과는 강화도에 비례
  영향:
    무기 강화도 1단계 증가
    → 특성 보너스 5% 증가
    → 기본 공격 데미지 약 2~3% 증가

강화 ← → 세트 효과
  연결: 같은 세트 장비를 강화
  의존: 세트는 강화된 장비와 함께 시너지
  영향:
    검 세트 4개 모두 +10 강화
    → 기본 공격 약 90% 증가 (세트 + 마스터리 + 강화)

강화 ← → 경제 시스템
  연결: 강화 비용 = 게임의 주요 골드 소비처
  의존: 플레이어는 강화 vs 다른 소비 선택
  영향:
    후반부: 월 골드 150,000 필요
    진행 속도: 강화 비용에 의해 결정

강화 ← → 성장 시스템
  연결: 강화 = 성장의 가장 큰 동기
  의존: 플레이어는 "다음 +1을 위해" 계속 플레이
  영향:
    강화 목표 = 성장의 주요 마일스톤
    "+10 완성" = 엔드게임 도달
```

## 4️⃣ 성장 시스템 (Progression System)

### 핵심 메커니즘

**성장의 작동 원리:**

```
성장 = 시간 경과에 따른 능력치 증가

레벨:
  범위: Lv 1 ~ Lv 100
  경험치: 100 × (Lv^1.4)
  
  Lv 1: 100 EXP
  Lv 50: 약 13,000 EXP
  Lv 100: 약 100,000 EXP
  누적: 약 1,000,000 EXP

특성 포인트:
  획득: 매 레벨업 +5pt
  누적: 500pt (Lv 100)
  분배: ATK, DEF, HP, SPD 중 선택

마일스톤:
  Lv 10: 고급 장비
  Lv 25: 새 카드 타입
  Lv 50: 새 지역
  등등...

결과:
  더 높은 레벨 = 더 강한 캐릭터
```

### 실제 게임플레이 흐름

```
[Lv 50, 경험치 8,500/13,000]

플레이:
  - 일일 던전 (30분)
  - 경험치 획득: 2,000 EXP
  
진행도:
  8,500 + 2,000 = 10,500/13,000 (80%)

계속 플레이:
  - 일일 던전 2차 (30분)
  - 경험치 획득: 2,500 EXP
  
[Lv 50 완료!]
  
현재: 10,500 + 2,500 = 13,000/13,000 (100%)
  ↓
[Lv 51 시작!]
  필요 EXP: 100 × (51^1.4) = 약 13,500 EXP
  
[레벨업 보상]
  특성 포인트 +5
  현재: 250pt (Lv 50) + 5pt = 255pt
  
[특성 포인트 분배]
  현재 빌드: Architect (공격형)
  추천:
    ATK: +3pt (총 60pt, 공격력 +60)
    DEF: +1pt (총 15pt, 방어력 +15)
    SPD: +1pt (총 12pt, 속도 +6)
  
  분배 후:
    ATK: 60 + 60 = 120 증가
    DEF: 15 + 15 = 30 증가
    최종 데미지: 이전 52.9 → 약 65 (+12)

[선택적: 마일스톤 달성]
  아직 Lv 25, 50은 지났으니까
  Lv 60까지 진행하면 특별 보상이...

이런 식으로 계속 성장
```

### 성장과 다른 시스템의 상호작용

```
성장 ← → 특성/세트/강화 모든 시스템
  연결: 성장 = 모든 시스템의 기초
  의존: 높은 레벨 = 더 많은 특성 포인트
  영향:
    레벨 = 게임 진행의 시간 척도
    모든 보너스는 레벨에 종속

성장 ← → 경제 시스템
  연결: 높은 레벨 = 더 높은 보상
  의존: 플레이어는 "레벨 올리기 위해" 던전 반복
  영향:
    Lv 1-20: 월 10시간
    Lv 50-70: 월 20시간
    Lv 80-100: 월 40시간 (극한)

성장 ← → 스토리
  연결: 스토리 진행 = 레벨 요구사항
  의존: 스토리 보스 = 특정 레벨 추천
  영향:
    막 1: Lv 1-5 권장
    막 2: Lv 10-30 권장
    막 5: Lv 70-100 권장
```

## 🔄 전체 시스템 통합 (System Integration)

### 데이터 흐름도

```
게임 시작
  ↓
[캐릭터 생성: Nox (Lv 1, ATK 15)]
  ├─ 기본 스탯 할당
  ├─ 기본 카드 5장 선택
  └─ 기본 장비 (무기, 방어구)
  ↓
[전투 시작]
  ├─ 카드 손패 생성 (5장)
  ├─ 매 턴마다 카드 선택
  └─ 기본 공격 자동 실행
       └─ 데미지 계산:
           1. 기본 데미지 (무기 ATK)
           2. 특성 적용 (마스터리)
           3. 세트 보너스 적용 (세트 진행도)
           4. 강화 보너스 적용 (강화도)
           5. 최종 데미지 계산
  ↓
[전투 승리]
  ├─ 경험치 획득 (Lv 업까지 남은 경험치 감소)
  ├─ 카드 획득 (무작위)
  ├─ 장비 획득 (무작위)
  └─ 골드 획득 (다음 강화비 축적)
  ↓
[레벨업 (특성 포인트 +5)]
  ├─ 특성 포인트 분배
  ├─ 특성 효과 변경
  └─ 다음 전투 데미지 증가
  ↓
[장비 관리]
  ├─ 새로운 장비 선택
  ├─ 세트 진행도 변경
  ├─ 특성 보너스 자동 계산
  └─ 기본 공격 데미지 즉시 반영
  ↓
[강화]
  ├─ 비용 소비
  ├─ 성공/실패 (강화도에 따라)
  ├─ 특성 레벨 변경
  ├─ 세트 시너지 증가
  └─ 최종 데미지 5~10% 증가
  ↓
[계속 플레이] → [전투]로 돌아가기

이 루프가 반복되면서:
  - Lv 1 → Lv 100으로 진행
  - 특성 포인트: 0 → 500
  - 강화도: 모든 슬롯 +0 → +10
  - 데미지: 15 → 150+ (10배 증가)
```

### 각 시스템의 역할

```
특성 시스템:
  역할: 장비의 효율성을 높여주는 "배수"
  플레이어의 선택: 장비 선택 (특성은 자동)
  영향: 모든 데미지 계산에 포함
  체감: 강화할 때마다 5% 증가

세트 효과:
  역할: 플레이 스타일을 강화해주는 "방향성"
  플레이어의 선택: 어떤 세트를 만들 것인가?
  영향: 특정 장비에만 보너스 (선택성)
  체감: 세트 완성될 때마다 크게 증가

강화 시스템:
  역할: 시간 투자의 보상 (경제의 주요 소비처)
  플레이어의 선택: 어떤 장비를 강화할 것인가?
  영향: 특정 장비의 성능 향상
  체감: 매번 5~10% 증가 (중독적)

성장 시스템:
  역할: 게임의 전체 진행도 척도
  플레이어의 선택: 어디에 특성 포인트를 쓸 것인가?
  영향: 모든 스탯의 기초
  체감: 매 레벨업마다 약 1~2% 증가
```

class CharacterTraits:
  weapon_mastery_level = equipment.weapon.enhancement_level

    return (weapon_mastery_level + 1) × 0.05  # +5% per level
  
  # 다른 마스터리도 동일하게...
  
  def apply_all_traits():
    base_damage = weapon.atk
    base_damage *= (1 + get_weapon_mastery_bonus())
    base_damage *= (1 + get_set_effect_bonus())
    base_damage *= (1 + get_accessory_synergy_bonus())
    return base_damage

def check_active_sets():
  for set_name in SET_DEFINITIONS:
    items_found = count_equipped_items_in_set(set_name)
    if items_found >= 2:
      apply_set_bonus(SET_DEFINITIONS[set_name].bonuses[items_found])

### 강화 시스템 구현

```
def calculate_enhancement_cost(level):
  return 1500 * (1.4 ^ level)

def attempt_enhancement(equipment, current_level):
  if current_level >= 10:
    return False  # Already max
  
  cost = calculate_enhancement_cost(current_level)
  success_rate = [100, 100, 100, 100, 100, 100, 85, 70, 55, 40][current_level]
  
  if randf() < success_rate:
    equipment.enhancement_level += 1
    return True
  else:
    if current_level > 0:
      equipment.enhancement_level -= 1
    return False
```
