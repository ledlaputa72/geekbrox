## 1. 시스템 개요

Dream Collector의 전투는 **전투 유형에 따라 두 시스템을 분리** 사용한다.

| 전투 유형 | 시스템 | 씬 | 특징 |
|----------|--------|-----|------|
| 일반 몬스터 | ATB (실시간) | CombatSceneATB | ATB 게이지 자동 진행, 리액션 타이밍 |
| 보스 | 턴베이스 | CombatSceneTB | 턴 단계, 에너지 전략, 보스 손패 유지 |

두 시스템은 **30장의 공통 카드 풀**을 공유한다. 같은 덱으로 완전히 다른 플레이 경험을 제공하는 것이 핵심 설계 철학이다.

## 2. ATB 전투 (일반 전투)

### 2.1 ATB 게이지

모든 캐릭터와 몬스터는 ATB 게이지를 보유하며 속도(SPD)에 따라 자동 충전된다.

```
ATB 충전식: ATB += (SPD / 100) per frame
ATB >= 1.0 → 행동 트리거 → ATB = 0 으로 리셋
```

**기본 스탯값**
- 플레이어 SPD: 70 (약 1.43초마다 기본 공격)
- 기본 ATB 최대값: 1.0 (정규화)

### 2.2 에너지 시스템

| 항목 | 값 |
|------|-----|
| 기본 에너지 | 3 |
| 에너지 최대 | 3 (오버플로우 시 최대 5) |
| 에너지 자동 회복 | +1 / 5초 |
| 오버플로우 지속 | 3.0초 |
| PASS 쿨다운 | 10초 |

### 2.3 ATB 전투 흐름

```
전투 시작
    ↓
ATB 게이지 실시간 충전 (항상 진행)
    ↓
캐릭터/몬스터 ATB 100% 도달
    ↓
자동 기본 공격 실행
    │
    └─→ 몬스터 ATB 100% 도달 시
            ↓
        "!" 아이콘 등장 (공격 예고)
        리액션 윈도우 오픈
            ↓
        플레이어 리액션 입력 (패링/회피/가드)
        또는 시간 초과 (기본 피해)
            ↓
        결과 적용 → ATB 리셋 → 반복
```

### 2.4 포커스 모드 / 크라이시스 모드

| 모드 | 발동 조건 | 효과 |
|------|----------|------|
| 포커스 모드 | 에너지 1 소모 수동 발동 | ATB 속도 0.3배, 에너지 10%/초 소모, 3초 지속 |
| 크라이시스 모드 | HP ≤ 30% | ATB 속도 0.5배, 10초 지속 |

## 3. 턴베이스 전투 (보스 전투)

### 3.1 턴 구조

```
플레이어 턴
    → 에너지 3 충전 (방어 카드 사용 시 다음 턴 +1 보너스)
    → 카드 사용 (에너지 소모)
    → 턴 종료 선언
        ↓
보스 공격 턴
    → 보스 행동 예고 2~3개 표시 (인텐트)
    → 보스 행동 실행
    → 리액션 윈도우 오픈 (에너지 무소모)
    → 플레이어 리액션
        ↓
적 모든 행동 종료 후
    → 플레이어 손패 버림 (discard)
    → 새 턴 시작
```

**중요**: 플레이어 손패는 **모든 적 행동이 끝난 뒤**에만 버려진다. 따라서 보스 공격 중 리액션에 카드를 사용할 수 있다.

### 3.2 턴베이스 에너지

| 항목 | 값 |
|------|-----|
| 기본 에너지 (턴 시작) | 3 |
| 방어 카드 사용 보너스 | 다음 턴 +1 |
| 보스 공격 중 리액션 | 에너지 무소모 |

### 3.3 타로 에너지 (보스전 전용)

주요 아르카나(MAJOR_ARCANA) 카드 사용 시 타로 에너지 충전. 특정 임계치 도달 시 특수 효과 발동.

### 3.4 꿈의 파편 (Dream Shard)

패링/회피 성공 시 획득. 특수 카드나 보스전 스킬 발동에 사용.

## 4. 리액션 시스템

### 4.1 개요

> **모바일 최적화 설계**: 프레임 단위 타이밍 대신, 색상 구간(zone) 기반 판정으로 모바일 터치 지연을 수용한다.

몬스터가 공격을 준비할 때 머리 위에 **"!" 아이콘** 하나가 등장하며, 이 아이콘의 **색상이 녹색 → 노란색 → 빨간색**으로 순차적으로 변한다. 아이콘은 한 번 나타난 뒤 사라지지 않고 색만 변한다(깜빡임 없음).

### 4.2 색상 구간과 판정

| 구간 | 색상 | 가능한 행동 | 의미 |
|------|------|------------|------|
| 1구간 (가장 김) | 🟢 녹색 | 가드만 가능 | "공격이 예고됐다" |
| 2구간 (중간) | 🟡 노란색 | 회피 + 가드 | "반응할 시간이다" |
| 3구간 (가장 짧음) | 🔴 빨간색 | 패링 + 회피 + 가드 | "지금이 패링 타이밍" |

**패링은 빨간색 구간에서만 성공한다** (가장 짧고 가장 어렵다).

### 4.3 타이밍 수치 (구현값)

| 모드 | 녹색 구간 | 노란색 구간 | 빨간색 구간 | 전체 |
|------|----------|------------|------------|------|
| Story | 1.0초 | 1.0초 | 0.4초 | 2.4초 |
| Hard | 0.65초 | 0.70초 | 0.25초 | 1.60초 |

### 4.4 리액션 버튼 동작

전투 하단에 **리액션 버튼 1개**가 표시된다. 리액션 윈도우가 열려있을 때만 활성화된다.

**버튼 카드 우선순위**: 손패에서 PARRY > DODGE > GUARD 순으로 1장 선택하여 버튼에 반영.

**연속 시도**: 패링을 잘못된 구간(녹색/노란색)에서 누르면 실패 처리 후 버튼이 회피 또는 가드로 자동 갱신된다. 같은 윈도우 안에서 다시 시도 가능.

### 4.5 결과 규칙

#### 성공 시

| 행동 | 데미지 | 추가 효과 |
|------|--------|-----------|
| **패링 성공** | 0 | 해당 몬스터 ATB 완전 초기화 (enemy.atb = -ATB_MAX) → 다음 행동까지 크게 느려짐 |
| **회피 성공** | 0 | 없음 |
| **가드** | 가드 수치만큼 경감, 나머지 HP 감소 | 없음 |

#### 실패 시 (잘못된 타이밍 또는 잘못된 선택)

| 행동 | 데미지 | 추가 효과 |
|------|--------|-----------|
| **패링 실패** | 기본 데미지 × 1.5 (+50%) | 해당 몬스터 다음 행동 빨라짐 (ATB = ATB_MAX × 0.5) |
| **회피 실패** | 기본 데미지 × 1.2 (+20%) | 없음 |

#### 미선택 (시간 초과)

기본 데미지 적용. 별도 패널티 없음.

### 4.6 방어 불가(UNBLOCKABLE) 공격

"!!" 아이콘으로 표시. 노란색(회피 구간) → 빨간색(패링 불가)으로 변한다.
이 공격에 패링을 시도하면 실패 처리 후 버튼이 회피로 전환된다.

### 4.7 로그 및 피드백

전투 로그: `패링 성공! (몬스터명)`, `패링 실패! (몬스터명) 피해 N (+50%) / 적 ATB 빨라짐` 등 몬스터명 포함.
플레이어 머리 위: `패링 성공!`, `회피 성공!`, `가드`, `패링 실패!`, `회피 실패!` 플로팅 텍스트 (색상 구분).

### 4.8 치명타(Critical Hit) 시스템

#### 개요
플레이어의 모든 공격(카드 기반)과 기본 공격에는 치명타 확률이 적용된다. 치명타는 데미지 계산의 **마지막 단계**에서 적용된다.

#### 치명타율(Crit Rate) 계산
```
플레이어 치명타율(%) = Base_CritRate(5%) + 장비_CRI_스탯
```

| 항목 | 값 |
|------|-----|
| 기본 치명타율 | 5% |
| 최대 치명타율 | 75% (하드 캡) |
| 장비에서 추가 가능 | 무기만 (반지/목걸이 불가) |

#### 치명타 판정 로직

```gdscript
# CombatManagerATB.gd 전투 중 예시 코드
var base_damage = player_atk * card_multiplier  # 기본 데미지

# 1. 치명타율 확인
var crit_chance = player_data.get("cri", 0.0)  # 장비 CRI 스탯 포함
var is_crit = randf() * 100.0 < crit_chance

# 2. 치명타 판정
if is_crit:
    damage = int(base_damage * 1.5)  # 150% 배율 적용
    battle_log("치명타! 데미지 %d" % damage)
else:
    damage = int(base_damage)

# 3. 최종 데미지 적용
monster.take_damage(damage)
```

#### 데미지 계산 순서 (전체)

```
Step 1: Base Damage = ATK × Card_Multiplier
        (예: 100 × 2.5 = 250)

Step 2: Apply Card Bonuses & Weapon Effects
        × (1 + Weapon_All_Eff% + Necklace_CardType%)
        (예: 250 × 1.3 = 325)

Step 3: Apply Status Effects (Vulnerable, Weakness, Strength 등)
        × 상태이상 보정값
        (예: 325 × 1.0 = 325)

Step 4: Apply Crit Damage
        IF is_crit:
            × 1.5  (치명타 배율)
        (예: 325 × 1.5 = 487.5)

Step 5: Apply Element Weakness/Resistance
        × elem_multiplier (약점 ×1.5, 저항 ×0.5)
        (예: 487.5 × 1.0 = 487.5)

Step 6: Monster Defense & Armor Penetration
        × (1 - DEF/(DEF+100)) × (1 - Armor_Penetration%)
        (예: 487.5 × 0.67 = 327)

Step 7: Final Damage Amplification
        Final = max(1, damage × Damage_Amplify%)
        (예: 327 × 1.0 = 327)
```

#### 치명타 피드백

| 요소 | 표현 | 위치 |
|------|------|------|
| 피해 숫자 | 🟡노란색 + 크기 1.2배 | 몬스터 머리 위 |
| 플로팅 텍스트 | "치명타!" | 피해 숫자 아래 |
| 전투 로그 | "치명타! 데미지 327" | 하단 로그 패널 |
| 사운드 | 크리티컬 히트 SFX (고음) | 스테레오 |
| VFX | 황색/적색 번쩍임 (0.3초) | 몬스터 주변 |

#### 장비 CRI 스탯 규칙

**무기만 CRI 스탯 보유 가능**:
```
무기 CRI 스탯 범위: 0~25% (희귀도별)
├─ COMMON:    0%
├─ UNCOMMON:  +5% (무기 기본)
├─ RARE:      +10~15%
├─ EPIC:      +15~20%
└─ LEGENDARY: +20~25%

장비 강화 시 CRI 증가:
├─ 강화 공식: base_cri + (level × 0.5%)
└─ 예: Lv10 무기 (base 15%) → 15% + 5% = 20%
```

**반지/목걸이는 CRI 스탯 불가**:
- ATK, DEF, HP, SPD만 보유
- 대신 카드 타입 효율 증가(목걸이) 또는 다른 버프 제공

#### 인게임 예시

```
[전투] 플레이어가 "불꽃 검" 공격!
       → 기본 데미지: 250
       → 치명타 발동! (10% 확률)
       → 최종 데미지: 375 (250 × 1.5)
       → 몬스터 HP: 625/1000
```

## 5. 카드 시스템

### 5.1 카드 구성 (30장 구현 + 42장 설계)

**M1 (현재 구현 상태)**: 30장
**M1+ (목표)**: 42장 (POWER 추가 6장 + CURSE 추가 6장)

| 타입 | 수량 | 특징 | 신 ID 체계 |
|------|------|------|----------|
| ATTACK (공격) | 10장 | 비용 1~4, 타로 테마 | SGL_, MLT_, SCL_, HYB_ |
| SKILL - GUARD (방어) | 8장 | 블록값, 일부 힐·드로우 | GRD_001~008 |
| SKILL - PARRY (패링) | 5장 | 비용 0~1, 리액션 전용, 에너지 보상 | PAR_001~005 |
| SKILL - DODGE (회피) | 5장 | 비용 0~1, 리액션 전용 | DOD_001~005 |
| POWER | 2장 (구현) + 6장 (설계) | 전투 전체 지속 버프 | ENR_, STR_, GRD_009, TRG_ |
| CURSE | 6장 (설계) | 적 디버프/약화 | WKN_, SPD_, PEN_, RSK_ |

## 6. 오토 AI

| 모드 | 동작 | 발동 조건 |
|------|------|----------|
| MANUAL | 플레이어가 모든 것을 직접 조작 | **기본값** (첫 5회 전투) |
| SEMI | 기본 공격만 자동, 카드/리액션은 수동 | 플레이어 설정 |
| FULL | 카드 포함 모든 것 자동 | 플레이어 설정 |

> **설계 의도**: 신규 유저가 패링/회피/콤보 시스템을 경험하지 못한 채 방치 게임으로 인식하는 것을 방지하기 위해 MANUAL이 기본값이다.

## 7. UI 시스템

### 7.1 액션 큐 (상단)

전투 화면 바로 위 얇은 바에 다음 행동 순서 최대 5개를 아이콘으로 표시.

- 아이콘 크기: 기본 대비 50% (15×15px)
- 현재 턴: ▲(흰색) 표시 + 2px 흰 테두리
- 발밑 타원 그림자: 현재 행동 중인 캐릭터/몬스터 표시

### 7.2 탐험 로그 (하단)

- 최대 3개 표시 (위 = 오래된 것, 아래 = 최신)
- 새 로그 추가 시 맨 위(오래된 것)부터 1개씩 삭제

### 7.3 피해/회복/블록 숫자

- 데미지: 빨간색 숫자 플로팅
- 체력 회복: 녹색 +숫자 플로팅
- 블록 증가: 파란색 🛡+숫자 플로팅

### 7.4 카드 애니메이션

- 카드 선택 → 대상(플레이어/몬스터)을 향해 날아가며 축소
- 이전 카드가 적용된 뒤 다음 카드 애니 시작 (순서 보장)
