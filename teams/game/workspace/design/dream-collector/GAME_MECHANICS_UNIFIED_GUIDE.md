**작성:** Atlas (PM) → Game팀 (개발 기준)  
**기준:** 특성/세트/강화/성장 시스템 통합  
**버전:** v1.0  
**상태:** 📋 컨셉 확정 및 개선 방안 제시

## 📌 개요

**목적:** 4가지 게임 메커니즘에 대한 기존 기획 정리 + 개선 방안 제시

**상태:**
- ✅ 특성 시스템: 기획 완료 (CHARACTER_TRAITS_ENHANCED.md)

## 1️⃣ 특성 시스템 (Trait/Mastery System)

### 기존 기획

**5가지 마스터리:**

```
1. 무기 마스터리 (Weapon Mastery)
   - 활성 조건: 무기 장착
   - 레벨: 무기 강화도 (+0~+10)
   - 효과: 무기 ATK +5%~+30%, 치명타 +1~10%
   - 특성 보너스: 검/도끼/지팡이 전문화
   
2. 방어 강화 (Defense Mastery)
   - 활성 조건: 방어구 장착
   - 레벨: 방어구 강화도
   - 효과: 방어구 DEF +5%~+30%, HP +5%~+30%
   - 특성 보너스: 갑옷/로브/경갑 전문화
   
3. 악세서리 동조 (Accessory Resonance)
   - 활성 조건: 악세서리 장착 (1~3개)
   - 레벨: 악세서리 개수
   - 효과: 악세서리 효과 100~150%, 시너지 +0~+20%
   - 특성 보너스: 반지/팔찌/장신구 조합
   
4. 듀얼 웰드 (Dual Wield)
   - 활성 조건: 보조 무기 장착
   - 레벨: 보조 무기 강화도
   - 효과: 추가 데미지 50~200%, 추가 공격 확률 +3~+20%
   - 특성 보너스: 방패/단검/보조주술서 타입
   
5. 장비 연쇄 (Equipment Chain)
   - 활성 조건: 같은 시리즈 장비 2개 이상
   - 레벨: 장착한 같은 시리즈 개수
   - 효과: 세트 보너스 +10%~+20%
   - 특성 보너스: 세트 완성 시 최대 효과
```

### OPS팀 검증 결과

```
점수: 98/100 (Grade A+)

평가:
  ✅ 5가지 마스터리가 명확하게 구분됨
  ✅ 각 마스터리가 다른 장비와 연동
  ✅ 재미와 복잡도 균형 좋음

발견된 문제: 없음

권장사항: 
  특성 UI에 다음 요소 추가
  - 현재 레벨 명시
  - 다음 레벨까지 진행도 표시
  - 각 마스터리가 기본 공격에 미치는 영향도 %로 표시
```

### Game팀용 개선 제안

#### 제안 1: 특성 포인트 시각화 개선

**현재:**
```
CharacterTraits.gd에서 각 마스터리를 개별 계산
UI에서는 최종 수치만 표시
```

**개선:**
```
특성 분해도 (Breakdown) 추가:
  기본 공격 = 35 데미지
  ├─ 무기 마스터리 (Lv 6): +30% = +10.5
  ├─ 악세서리 동조 (Lv 3): +20% = +7
  ├─ 듀얼 웰드 (Lv 6): +200% = +70
  └─ 장비 연쇄 (4/4): +20% = +7
  ────────────────────────
  최종 데미지: 65.5 (87% 증가)

UI에 "특성 분해" 버튼으로 상세 표시
```

#### 제안 2: 마스터리별 목표 단계 추가

**현재:**
```
특성 레벨은 자동으로 증가
플레이어는 수동으로 달성할 목표 불명확
```

**개선:**
```
각 마스터리별 마일스톤:
  무기 마스터리:
    Lv 1: 무기 획득 (자동)
    Lv 3: 무기 +3 강화 (목표)
    Lv 6: 무기 +10 강화 (최종)
    
  방어 강화:
    Lv 1: 방어구 획득
    Lv 3: 방어구 +3 강화
    Lv 6: 방어구 +10 강화
  
이렇게 하면 플레이어가 "다음 목표"를 명확히 알 수 있음
```

#### 제안 3: 마스터리 조합 권장 시스템

**현재:**
```
5가지 마스터리가 독립적으로 작동
```

**개선:**
```
플레이 스타일별 권장 마스터리 조합:

Architect (공격형):
  추천: 무기(Lv6) + 듀얼(Lv6) + 악세(Lv1)
  제안 문구: "쌍검 전사 빌드" → 추천 강화 경로 표시

Healer (방어형):
  추천: 방어(Lv6) + 무기(Lv1) + 악세(Lv3)
  제안 문구: "강철 수호자 빌드"

Mystic (균형형):
  추천: 모든 것 Lv 3
  제안 문구: "전술 마스터 빌드"

이를 통해 새 플레이어도 강화 방향을 쉽게 이해 가능
```

## 2️⃣ 세트 효과 (Set Effect System)

### 기존 기획

**4가지 프리정의 세트:**

```
검 세트 (검 + 갑옷 + 반지 + 방패):
  2개: ATK +10%, 전투력 +5%
  3개: ATK +15%, DEF +10%
  4개: ATK +20%, DEF +15%, 기본 공격 2회 연속

마법 세트 (지팡이 + 로브 + 신발 + 주술서):
  2개: 마법 데미지 +15%, 공격 속도 +10%
  3개: 마법 데미지 +25%, 공격 속도 +20%
  4개: 마법 데미지 +35%, 공격 속도 +30%, 매턴 추가 공격

방어 세트 (도끼 + 갑옷 + 반지 + 방패):
  2개: DEF +15%, HP +5%
  3개: DEF +25%, HP +10%
  4개: DEF +35%, HP +15%, 매턴 5% 회복

회피 세트 (지팡이 + 경갑 + 반지 + 신발):
  2개: 회피율 +15%, 공격 속도 +15%
  3개: 회피율 +25%, 공격 속도 +25%
  4개: 회피율 +35%, 공격 속도 +35%, 기본 공격 시 회피 +50%

자유 조합:
  같은 타입 3개: 기본 스탯 +30%
  다양한 타입 4개: 모든 스탯 +15%
```

### OPS팀 검증 결과

```
점수: 92/100 (Grade A)

평가:
  ✅ 4가지 세트가 각각 다른 플레이 스타일 대표
  ✅ 플레이어가 목표를 명확히 설정 가능
  ✅ 자유 조합도 존재하여 유연성 있음

발견된 문제:
  세트 효과 표시 UI가 복잡할 수 있음
  일부 플레이어가 세트 진행도를 모를 수 있음

권장사항:
  1. 세트 진행도 표시 (2/4 완성 등)
  2. 다음 세트 장비 추천
  3. 세트 효과 시각화 개선
```

### Game팀용 개선 제안

#### 제안 1: 세트 진행도 UI 추가

**현재:**
```
장비 화면에는 개별 장비만 표시
세트 진행도는 별도 확인 필요
```

**개선:**
```
장비 화면에 "세트 진행도" 탭 추가:

검 세트:
  [검 ✓] [갑옷 ✓] [반지 ✗] [방패 ✓]
  진행도: 3/4
  다음 필요: 반지 (권장: 왕의 반지)
  4개 완성 시 보너스: ATK +20%, DEF +15%, 2회 연속
  
이렇게 하면 플레이어가 "뭐가 필요한지" 한눈에 파악 가능
```

#### 제안 2: 세트 효과 예측 시스템

**현재:**
```
세트 효과는 고정되어 있음
플레이어는 미리 어떤 효과가 있을지 알기 어려움
```

**개선:**
```
세트 클릭 시 "효과 미리보기":

검 세트 (2/4):
  ├─ 현재 활성: ATK +10%, 전투력 +5%
  ├─ 3개 시 추가: DEF +10% (추가 예정)
  └─ 4개 시 추가: 기본 공격 2회 연속 (초대 예정)
  
이를 통해 플레이어가 "3개 모아야 DEF 보너스를 얻을 수 있다"는 것을 인식
```

#### 제안 3: 세트 추천 알고리즘

**현재:**
```
세트는 고정되어 있고 선택은 플레이어 몫
```

**개선:**
```
캐릭터 분석 후 세트 추천:

현재 Nox (Architect, 검 3개 착용):
  추천 세트: "검 세트 (3/4)"
  → "왕의 반지 1개만 더 모으면 
     ATK +20%, DEF +15% 추가 보너스 획득 가능합니다!"
  
버튼: [왕의 반지 찾기] → 일일 던전 추천

이를 통해 신규 플레이어도 세트 목표를 자연스럽게 설정
```

#### 제안 4: 추가 세트 제안 (후속 패치)

**현재 세트 4개:**
```
검, 마법, 방어, 회피 (4가지)
```

**추가 세트 제안 (2.0 버전):**
```
1. 광전사 세트 (검 + 도끼 조합)
   - 극도의 공격성
   - ATK +50%, DEF -20% (위험)

2. 암살자 세트 (단검 + 경갑 + 신발 조합)
   - 높은 회피 + 빠른 공격
   - 회피 +40%, 공격 속도 +40%

3. 지원가 세트 (로브 + 신발 + 악세서리 조합)
   - 팀 게임 대비 (향후 Co-op)
   - 동료 버프 +20%

4. 위장 세트 (경갑 + 신발 + 장신구 조합)
   - 혼자 몰래 진행하는 느낌
   - 적 감지율 -50%, 피해 -20%

5. 사제 세트 (지팡이 + 로브 + 목걸이 조합)
   - 신성한 힘
   - 신성 데미지 +30%, 아군 회복 +20%

이렇게 9가지 세트로 확장하면 플레이 스타일 다양성 증가
```

## 3️⃣ 강화 시스템 (Enhancement System)

### 기존 기획

```
강화 단계: +0 ~ +10
기본 비용: 1,000 골드 × 1.5^강화도

강화 단계별 성공률:
  +0~+6: 100% (항상 성공)
  +7: 85% (실패 시 단계 하락)
  +8: 70%
  +9: 55%
  +10: 40%

효과: 각 단계마다 스탯 +5~10% 증가
```

### OPS팀 검증 결과

```
점수: 92/100 (Grade A)

평가:
  ✅ 강화 시스템의 중독성 높음
  ✅ 위험도와 보상의 균형 좋음
  ✅ 진행도 느끼기 좋음

발견된 문제:
  ⚠️ 후반부 강화 비용 과도 (1.5배 배수)
  ⚠️ 실패 시 좌절감 높음 (+7부터 위험)

권장 개선:
  1. 기본값: 1,000 G → 1,500 G (상향)
  2. 배수: 1.5배 → 1.4배 (하향)
  3. 강화 보호 시스템 추가
  4. 강화 재료 시스템 추가
```

### Game팀용 개선 제안

#### 제안 1: 강화 보호 시스템 (Protection System)

**현재:**
```
실패하면 무조건 단계 하락
+9 → +8로 떨어지는 상황 가능
```

**개선 옵션 1: 보호권 시스템**
```
매월 무료 보호권 2개 제공:
  +7, +8, +9 강화 시 보호권 소비로 실패 방지
  
무료: 월 2개
유료: 100 다이아 = 1개 보호권

이를 통해:
  - 무료 플레이어도 위험감소
  - 유료 플레이어도 추가 소비 기회
```

**개선 옵션 2: 안전 강화 (Safe Enhancement)**
```
강화 시 2가지 선택:
  1. 일반 강화 (현재): 40% 실패율 (-1 단계)
  2. 안전 강화: 100% 성공, 비용 2배
  
선택지를 주면 플레이어가 전략적으로 선택
```

#### 제안 2: 강화 재료 시스템

**현재:**
```
강화 비용은 순수 골드만 소비
```

**개선:**
```
골드 + 강화 재료 시스템:

강화 재료 획득:
  - 일일 던전 드롭 (확률)
  - 장비 분해 (새로운 기능)
  - 이벤트 보상

강화 요구 재료:
  +0~+5: 골드만
  +6~+7: 골드 + 강화석 (1개)
  +8~+9: 골드 + 강화석 (2개) + 영혼 결정 (1개)
  +10: 골드 + 강화석 (3개) + 영혼 결정 (2개) + 궁극의 조각 (1개)

효과:
  - 후기 경제에 목표 부여
  - 일일 던전 플레이 동기 증가
  - 강화 과정이 더 전략적
```

#### 제안 3: 강화 성공률 시각화 개선

**현재:**
```
강화 화면에 "성공률: 40%" 정도만 표시
```

**개선:**
```
더 상세한 정보 제공:

[+9 → +10 강화]
  성공률: 40% (2/5 확률)
  
  성공 시:
    ✓ 데미지 +10% 증가
    ✓ 전체 공격력: 62 → 68
  
  실패 시 (60%):
    ✗ 강화도 +9 → +8로 하락
    ✗ 금액 손실: 38,445 G
    ✓ 보호권 2개 남음

  [강화하기] [보호권 사용] [취소]

이렇게 하면 플레이어가 위험을 인식하고 선택 가능
```

## 4️⃣ 성장 시스템 (Progression System)

### 기존 기획

```
레벨: Lv 1 ~ Lv 100

경험치 공식:
  필요 EXP = 100 × (Lv^1.5)
  
  Lv 1: 100 EXP
  Lv 50: 35,355 EXP
  Lv 100: 100,000 EXP
  누적: 1,000,000 EXP

특성 포인트:
  매 레벨업: 5 포인트
  누적: Lv 100 시 500 포인트
  
  분배 대상:
    - 공격력 (ATK): 1pt = +1 ATK
    - 방어력 (DEF): 1pt = +1 DEF
    - 생명력 (HP): 1pt = +5 HP
    - 속도 (SPD): 1pt = +0.5 SPD
    - 특수 스킬 언락 (10~50pt)

마일스톤:
  Lv 5: 특별 스킬 포인트 +1
  Lv 10: 고급 장비 획득
  Lv 25: 새로운 카드 타입 언락
  Lv 50: 새로운 지역 오픈
```

### OPS팀 검증 결과

```
점수: 93/100 (Grade A)

평가:
  ✅ 레벨 곡선이 자연스러움
  ✅ 특성 포인트 분배 자유도 높음
  ✅ 마일스톤이 명확함

발견된 문제:
  ⚠️ 후반부 경험치 수집이 느림 (Lv 80-100)
  ⚠️ 특성 포인트 리셋이 없어서 실수 방지 불가
  ⚠️ 마일스톤이 너무 느린 주기 (25 Lv마다)

권장 개선:
  1. 후반부 경험치 수집 경로 추가
  2. 특성 포인트 리셋 기능 추가 (유료)
  3. 마일스톤 추가 (5 Lv마다)
  4. 특성 포인트 추천 시스템
```

### Game팀용 개선 제안

#### 제안 1: 마일스톤 확대 (기본 + 추가)

**현재:**
```
Lv 5, 10, 25, 50 (4개만)
```

**개선:**
```
마일스톤을 10 Lv마다 추가:

Lv 5: 특별 스킬 언락
Lv 10: 고급 장비 획득
Lv 15: 새로운 카드 5장 선택
Lv 20: 특성 포인트 +10 보너스
Lv 25: 새로운 카드 타입 언락
Lv 30: 마스터리 달성 (무기)
Lv 35: 새로운 카드 5장 선택
Lv 40: 특성 포인트 +10 보너스
Lv 45: 세계 확장 (새 지역)
Lv 50: 새로운 지역 오픈
Lv 55: 마스터리 달성 (방어)
...
Lv 100: 최종 칭호 획득

이렇게 하면 게임 진행이 더 물흐르듯 진행됨
```

#### 제안 2: 특성 포인트 추천 시스템

**현재:**
```
특성 포인트 분배는 완전히 플레이어 자유
신규 플레이어는 어떻게 분배해야 할지 모름
```

**개선:**
```
플레이 스타일별 추천:

현재 Nox (Architect):
  추천 빌드: "공격 특화"
  
  추천 분배 (Lv 50, 250pt):
    ATK: 150pt (60%)
    DEF: 50pt (20%)
    HP: 30pt (12%)
    SPD: 20pt (8%)
    
  효과: ATK +150, DEF +50, HP +150, SPD +10
  
  [추천대로 분배] [직접 분배] [미리보기]

또한 "리셋" 기능 추가 (100 다이아):
  실수로 분배한 것 취소 가능
```

#### 제안 3: 특성 포인트 보너스 확대

**현재:**
```
기본: 5pt / 레벨
보너스: 5 Lv마다 +1pt (Lv 5, 10, 15...)
```

**개선:**
```
보너스를 더 자주 제공:

기본: 5pt / 레벨

마일스톤 보너스:
  Lv 10: +10pt
  Lv 20: +10pt
  Lv 30: +10pt
  Lv 40: +10pt
  Lv 50: +20pt (마스터 달성)
  ...
  Lv 100: +50pt (최종 달성)

누적:
  기본: 500pt
  마일스톤: +200pt
  ──────────
  총: 700pt (40% 증가)

이를 통해:
  - 플레이어가 특성 분배에 더 자유로움
  - 마일스톤의 가치 증가
```

#### 제안 4: 특성 초기화 및 리스펙 시스템

**현재:**
```
특성 포인트 분배는 영구적
실수하면 복구 불가
```

**개선 옵션 1: 전체 리셋 (유료)**
```
전체 특성 포인트 초기화:
  비용: 100 다이아 (한 번에 최대 2회)
  
이를 통해:
  - 플레이어가 실수를 두려워하지 않음
  - 게임 진행 중 빌드 변경 가능
```

**개선 옵션 2: 부분 리셋 (무료)**
```
매달 1회 무료 부분 리셋:
  특정 스탯 1개만 초기화
  
예: ATK에 160pt 분배했는데, 50pt만 돌려받기
```

#### 제안 5: 플레이 시간별 경험치 곡선 조정

**현재:**
```
경험치 = 100 × (Lv^1.5)
선형적 증가
```

**OPS팀 피드백:**
```
Lv 80-100: 경험치 수집 너무 느림
월 100시간 플레이해도 4개월 소요
```

**개선:**
```
경험치 공식 조정:
  100 × (Lv^1.4) (1.5에서 1.4로)
  
효과:
  - Lv 80-100 속도 15% 증가
  - 극한 게이머: 월 1 Lv 진행 (현재 0.5 Lv)
  - 시간 투자에 대한 보상 증가
```
