## 1. 시스템 개요 — "방어 = 에너지 수확"

### 1.1 핵심 아이디어

기존 StS 방식의 문제: 적 턴은 그냥 맞거나 블록 카드를 미리 쌓는 것.
적 턴이 **자원 획득 기회**가 되도록 재설계.

```
[기존 StS 흐름]
플레이어 턴 → 적 턴 (그냥 맞음) → 다음 플레이어 턴

[Dream Collector 리액션 흐름]
플레이어 턴 → 적 턴 (방어 선택!) → 에너지 보너스 → 다음 플레이어 턴
                    ↓
            패링 > 회피 > 방어 > 무반응
            (에너지 보상 내림차순)
```

### 1.2 에너지 회복 구조

**기본 에너지**: 매 턴 시작 시 3 자동 충전 (StS 원칙 유지)

**방어 리액션 보너스 에너지**: 적 공격에 반응한 방어 행동으로 추가 획득
→ 다음 플레이어 턴 시작 시 기본 에너지에 합산

```
[에너지 구조]
다음 턴 에너지 = 기본 3 + 방어 보너스

패링 성공:   3 + 2 = 최대 5에너지
회피 성공:   3 + 1 = 4에너지
방어 성공:   3 + 0 = 3에너지 (기본값 유지)
무반응:      3 + 0 = 3에너지 (패널티 없음)
```

> 설계 의도: 패널티 없이 보상만 차등 부여 → 실패해도 불쾌하지 않음.
> 잘하면 더 많이 할 수 있는 구조.

## 2. 3가지 방어 행동 상세 설계

### 2.1 패링 (Parry / 쳐내기) ⭐ 최고 보상

**개념**: 적 공격이 닿는 순간 정확한 타이밍에 패링 카드를 탭.
소울류의 "튕겨내기"를 카드 게임으로 재해석.

| 항목 | 내용 |
|------|------|
| **타이밍 윈도우** | 0.5초 (공격 직전) |
| **피해 감소** | 100% (완전 무효) |
| **에너지 보너스** | +2 (다음 턴) |
| **추가 보상** | 카드 1장 즉시 드로우 |
| **난이도** | 높음 |

**패링 발동 조건**:
```
손패에 "패링" 태그가 붙은 카드 보유
+ 적 공격 직전 0.5초 윈도우 내 탭
→ 패링 성공
```

**패링 성공 연출**:
```
[패링 성공!]
- 짧은 흰색 플래시 + "쳐냈다!" 텍스트
- 에너지 +2 획득 알림 ("⚡+2 다음 턴")
- 즉시 카드 1장 드로우
- 0.2초 슬로우 (성취감 강화)
```

**패링 카드 목록** (초기 5종):

| 카드명 | 에너지 비용 | 효과 |
|--------|-----------|------|
| 꿈의 쳐내기 | 0 | 패링 전용 — 공격 무효 + ⚡+2 (다음 턴) + 드로우 1 |
| 반사의 순간 | 0 | 패링 전용 — 공격 무효 + ⚡+2 + 받을 피해 30% 반격 |
| 각성의 쳐내기 | 0 | 패링 전용 — 공격 무효 + ⚡+3 (다음 턴) (타이밍 더 좁음 0.3초) |
| 달빛 반격 | 1 | 패링 전용 — 공격 무효 + ⚡+1 + 즉시 공격 8 반격 |
| 완벽한 방어 | 0 | 패링/회피 겸용 — 공격 무효 + ⚡+1 |

---

### 2.2 회피 (Dodge / 피하기) ⭐⭐ 중간 보상

**개념**: 패링보다 넓은 윈도우. 데미지를 일부 또는 전부 피하되, 보상은 패링보다 낮음.

| 항목 | 내용 |
|------|------|
| **타이밍 윈도우** | 1.2초 (공격 예고 후) |
| **피해 감소** | 완전 회피 (타이밍 내 탭) |
| **에너지 보너스** | +1 (다음 턴) |
| **추가 보상** | 없음 |
| **난이도** | 중간 |

**회피 카드 목록** (초기 5종):

| 카드명 | 에너지 비용 | 효과 |
|--------|-----------|------|
| 꿈의 스텝 | 0 | 회피 전용 — 공격 회피 + ⚡+1 (다음 턴) |
| 잔상 (殘像) | 0 | 회피 전용 — 공격 회피 + ⚡+1 + 버프 1개 이전 |
| 황혼의 도약 | 0 | 회피 전용 — 공격 회피 + ⚡+1 + 다음 공격 +3 데미지 |
| 연막 | 1 | 회피 전용 — 공격 회피 + ⚡+1 + 다음 적 공격 데미지 -3 |
| 반보 앞으로 | 0 | 회피/방어 겸용 — 공격 50% 감소 + ⚡+1 |

---

### 2.3 방어 (Guard / 막기) ⭐⭐⭐ 기본 보상

**개념**: 손패의 방어 카드를 미리 또는 공격 중 사용. 블록 생성.
타이밍 불필요. StS의 기본 블록 방식.

| 항목 | 내용 |
|------|------|
| **타이밍 윈도우** | 없음 (플레이어 턴에 미리 사용) |
| **피해 감소** | 블록값만큼 감소 |
| **에너지 보너스** | +0 (기본 3 유지) |
| **추가 보상** | 없음 (블록 자체가 보상) |
| **난이도** | 낮음 |

> 방어 카드는 에너지 보너스는 없지만 **블록값 자체가 강력**하도록 밸런싱.
> "패링/회피를 못 하는 플레이어도 방어 카드로 충분히 진행 가능"

---

### 2.4 무반응 (No Action) — 패널티 없음

타이밍 윈도우 내 아무것도 안 하면 → 풀 데미지, 에너지 보너스 없음.
오토 플레이 시 AI가 자동으로 가장 적합한 방어 행동 선택.

## 3. 에너지 차등이 만드는 전략 경제

### 3.1 턴당 에너지 시나리오 비교

| 플레이 스타일 | 평균 에너지/턴 | 플레이 가능 카드 |
|------------|------------|-------------|
| 패링 마스터 | 4.5~5 | 카드 3~5장 |
| 회피 위주 | 3.5~4 | 카드 2~4장 |
| 방어 위주 | 3 | 카드 2~3장 |
| 무반응 | 3 | 카드 2~3장 |

→ 패링을 잘하는 플레이어는 같은 덱으로 더 많은 카드를 사용. 동일 덱 내 숙련도 차이가 성능으로 직결.

### 3.2 에너지 보너스의 전략적 활용

```
[패링 성공 → 에너지 5 상황]
일반 플레이어: 코스트 1~2짜리 카드 3~4장
숙련 플레이어: 코스트 3 강력 카드 + 코스트 2 카드 조합
→ 같은 패링 보상으로도 덱 설계에 따라 활용도 차이
```

**고비용 카드 설계 방향**: 패링 자주 하는 플레이어를 위해 에너지 4~5를 요구하는 강력한 카드 추가.

```
[패링 전용 고비용 카드 예시]
"꿈의 폭발" — 에너지 4 — 40 데미지 + 전체 디버프 제거
"달의 심판" — 에너지 5 — 60 데미지 (패링 성공 횟수 × +5)
```

## 4. 카드 획득 & 운영에서의 방어 차이

### 4.1 덱빌딩에서의 방어 특화

방어 스타일에 따라 **덱의 방향성**이 달라지도록 설계:

```
[패링 특화 덱]
├── 패링 카드 다수 보유
├── 고비용 강력 카드 포함 (패링 에너지 활용)
├── 빠른 공격 집중 (에너지 여유)
└── 드로우 카드 적게 필요 (패링 드로우로 보완)

[회피 특화 덱]
├── 회피 카드 + 버프 카드 조합
├── 회피 후 강화 카드 연계
├── 중간 비용 카드 효율 극대화
└── 상태이상 카드 집중 (회피로 생존하며 누적)

[방어 특화 덱]
├── 블록 카드 다량 보유 (분당 블록 최대화)
├── 블록 관련 파생 카드 (블록×공격력 등)
├── 저비용 카드 다수 (에너지 3 내에서 최대 수)
└── 안정적인 방어선 + 소량 공격
```

### 4.2 전투 보상 차등 (전투 종료 시)

방어 품질에 따라 **전투 후 카드 보상**에 소폭 차이:

| 방어 품질 | 카드 선택지 | 특이사항 |
|---------|-----------|---------|
| 패링 3회+ | 4장 중 선택 | 희귀(Rare) 카드 1장 보장 |
| 패링 1~2회 or 회피 다수 | 3장 중 선택 | 기본 |
| 방어 위주 (패링 0회) | 3장 중 선택 | 기본 |

→ 소폭 차이로 "패링이 유리"는 인식 주되, 강제는 하지 않음.

## 5. 의도(Intent) 시스템 — 턴베이스 버전

### 5.1 기본 구조

```
[몬스터 의도 표시 — 턴베이스]

꿈의 악몽
HP: ████████░░ 78%
─────────────────────────
다음 행동: ⚔️ 18    예정: 🛡️   그 다음: ⚔️ 25 ⚠️
           ↑ 이번 턴   ↑ 다음 턴    ↑ 두 턴 후
─────────────────────────
패링 가능: ✅   회피 가능: ✅   방어: ✅
```

### 5.2 공격 유형 표기

| 아이콘 | 의미 | 권장 대응 | 에너지 보너스 |
|--------|------|---------|------------|
| ⚔️ (숫자) | 일반 공격 | 패링 or 회피 or 방어 | 선택에 따라 차등 |
| ⚔️ ⚠️ | 강한 공격 (데미지 높음) | 패링 or 방어 필수 | 패링 시 +2 강조 |
| 🔱 (방어불가) | 관통 공격 | 회피 전용 | 회피 시 +1 |
| 🛡️ | 방어 강화 | 디버프 or 강공격 | 해당 없음 |
| ✨ | 버프 | 버프 제거 카드 | 해당 없음 |
| 💤 | 쉬는 턴 | 공격 집중 | 해당 없음 |

### 5.3 공격 예고 연출 — 턴베이스용 (단순화)

소울류처럼 복잡한 3단계 연출 대신 **2단계**로 단순화:

```
[Phase 1] 의도 표시 (플레이어 턴 내내)
  → 적 HP 바 위에 아이콘 + 데미지 숫자 표시
  → 패링/회피/방어 가능 여부 표시

[Phase 2] 공격 실행 직전 (적 턴 시작 시)
  → 0.8초 경고 팝업: "⚔️ 꿈의 악몽이 공격한다!"
  → 손패의 패링/회피 카드 황금 테두리 강조
  → 1.2초 리액션 윈도우 카운트다운 (UI 진행 바)
```

```gdscript
# TurnBasedIntentSystem.gd

func on_enemy_attack_begin(enemy: Enemy, attack: AttackData):
    # Phase 2 연출
    IntentUI.flash_warning(enemy.name, attack)
    HandUI.highlight_reaction_cards(attack.type)
    ReactionTimer.start(1.2)  # 1.2초 윈도우

func on_reaction_window_closed():
    # 윈도우 종료 → 방어 결과 적용
    match ReactionResult.current:
        "PARRY":  _apply_parry_result()
        "DODGE":  _apply_dodge_result()
        "GUARD":  _apply_guard_result()
        "NONE":   _apply_full_damage()
```

## 6. 오토 플레이 연동

### 6.1 오토 AI의 방어 선택 로직

```gdscript
# TurnBasedAutoAI.gd — 방어 행동 결정

func decide_defense(hand: Array[Card], attack: AttackData) -> String:
    # 1. 패링 카드 있으면 → 패링 시도 (75% 성공률로 AI 설정)
    if hand.has_parry_card() and attack.is_parryable:
        if randf() < 0.75:
            return "PARRY"

    # 2. 회피 카드 있으면 → 회피
    if hand.has_dodge_card() and attack.is_dodgeable:
        return "DODGE"

    # 3. 블록 카드로 방어
    if hand.has_guard_card():
        return "GUARD"

    # 4. 아무것도 없으면 → 무반응
    return "NONE"
```

**오토 패링 효율**: 수동의 75%. 즉 수동으로 잘 하면 더 많은 에너지 획득 가능 → 수동 플레이 인센티브 유지.

## 8. 전투 흐름 요약

```
[플레이어 턴]
에너지 = 3 + (지난 적 턴 방어 보너스)
카드 드로우 5장 (+ 지난 패링 드로우)
    ↓
카드 플레이 (에너지 소모)
    ↓
턴 종료
    ↓
[적 턴]
의도 표시 → 공격 시작
    ↓
[리액션 윈도우 1.2초]
  패링 탭 (0.5초 이내)   → 무효 + ⚡+2 + 드로우1
  회피 탭 (1.2초 이내)   → 회피 + ⚡+1
  방어 카드 탭           → 블록 + ⚡+0
  무반응               → 풀 피해 + ⚡+0
    ↓
[다음 플레이어 턴 시작]
에너지 = 3 + 방어 보너스 (위에서 결정됨)
```
