## 1. 시스템 개요 — ATB에서 방어의 의미

### 1.1 ATB 에너지 구조

ATB 전투에서 에너지는 두 가지 경로로 충전됨:

```
[에너지 충전 경로]

경로 A — 자동 충전 (시간 기반)
  에너지 타이머 만충 → 에너지 +1 자동 획득 (5초마다)
  손패 크기에 따라 회복 속도 소폭 조정

경로 B — 방어 리액션 보너스 (즉시 충전)
  패링 성공 → 에너지 즉시 +2
  회피 성공 → 에너지 즉시 +1
  방어 성공 → 에너지 즉시 +0.5 (절반 충전)

[에너지 최대치]
기본: 3
패링 후: 최대 5까지 일시적 초과 허용 (2초간, 이후 3으로 조정)
```

### 1.2 "즉시 충전"이 만드는 플레이 감각

```
[ATB 전투 순간]

적 ATB 80% → 공격 예고! → 플레이어 패링 탭
          ↓
    ⚡ 에너지 +2 즉시!
          ↓
    카드 2장 연속 플레이 가능
          ↓
    → 막자마자 반격! 라는 박진감
```

→ 방어와 공격이 하나의 연속 동작으로 연결됨. 소울류의 "패링 → 리포스트"를 ATB 카드로 표현.

## 2. 3가지 방어 행동 — ATB 버전

### 2.1 패링 (Parry / 쳐내기) — 즉각 에너지 폭발

**개념**: 적 ATB가 80% 이상일 때(공격 임박) 패링 카드를 탭.
ATB가 100% 도달해 공격 직전 0.5초 이내 탭 시 완전 패링 성공.

| 항목 | 내용 |
|------|------|
| **발동 윈도우** | 적 ATB 80%~100%, 임팩트 0.5초 전 |
| **피해 감소** | 100% (완전 무효) |
| **에너지 즉시 충전** | **+2 (즉시)** |
| **추가 보상** | 카드 1장 즉시 드로우 |
| **ATB 속도 효과** | 적 ATB 50% 롤백 (공격 지연) |

**패링 직후 흐름**:
```
패링 성공
  → ⚡⚡ 에너지 +2 즉시 (HUD 에너지 바 충전 애니메이션)
  → 카드 1장 드로우 (패에 추가)
  → 적 ATB 50%로 되돌아감 (공격 지연)
  → 플레이어: 충전된 에너지로 바로 카드 2장 플레이 가능!
```

**ATB 중 패링의 의미**:
- 방어하는 동시에 공격 템포를 앞당기는 이중 효과
- 패링 잘 할수록 적이 공격도 적게 하고, 내가 카드도 더 많이 씀

---

### 2.2 회피 (Dodge / 피하기) — 즉각 에너지 충전

**개념**: 패링보다 넓은 윈도우. 적 ATB 80% 이후 1.2초 이내 회피 카드 탭.

| 항목 | 내용 |
|------|------|
| **발동 윈도우** | 적 ATB 80%~공격 후 0.4초 |
| **피해 감소** | 완전 회피 (윈도우 내 탭) |
| **에너지 즉시 충전** | **+1 (즉시)** |
| **추가 보상** | 없음 |
| **ATB 속도 효과** | 없음 |

**회피 직후 흐름**:
```
회피 성공
  → ⚡ 에너지 +1 즉시
  → 플레이어: 충전된 에너지로 카드 1장 추가 플레이 가능
  → 전투 흐름 끊기지 않음 (패링보다 낮지만 부드럽게 연속)
```

---

### 2.3 방어 (Guard / 막기) — 에너지 소량 충전

**개념**: 손패의 방어 카드를 언제든 사용. 블록을 쌓아 피해를 흡수.
타이밍 불필요. 에너지 보너스는 적지만 블록 자체가 보상.

| 항목 | 내용 |
|------|------|
| **발동 타이밍** | 언제든 (ATB 진행 중) |
| **피해 감소** | 블록값만큼 흡수 |
| **에너지 충전** | **+0.5 즉시** (절반 충전, 2회면 +1) |
| **추가 보상** | 높은 블록값 카드가 보상 |

> ATB에서 방어 카드를 쓰면 **에너지 0.5**가 즉시 올라옴.
> 2번 방어 카드를 쓰면 에너지 +1. 방어를 많이 하는 플레이어도 충전 속도 유지.

---

### 2.4 무반응 (Miss) — 패널티 없음

리액션 윈도우 내 아무 행동 안 함 → 풀 피해, 에너지 보너스 없음.
오토 플레이 시 AI가 자동으로 회피/방어 중 선택.

## 3. 즉시 에너지 충전의 전략적 활용

### 3.1 "막고 바로 반격" 콤보 시나리오

```
[시나리오: 에너지 1 남은 상황]

에너지: ⚡ (1/3)
손패: [공격-2] [공격-2] [패링-0] [회피-0]

→ 적 ATB 80% 도달! 공격 예고

[패링 성공] → 에너지 즉시 +2 → 에너지 3/3
              카드 드로우 1장 → 손패에 추가

→ 바로 공격-2 + 공격-2 연속 사용!
→ "막자마자 풀카드 반격!"
```

```
[시나리오: 에너지 2 남은 상황]

에너지: ⚡⚡ (2/3)
→ 적 ATB 80% 도달!

[회피 성공] → 에너지 +1 → 에너지 3/3
→ 공격-2 + 공격-1 연속 사용 가능
```

### 3.2 에너지 "초과 충전" 시스템

패링 성공 시 에너지가 3을 초과할 수 있음 (최대 5, 2초 유지):

```gdscript
# ATBEnergySystem.gd
const ENERGY_MAX = 3
const ENERGY_OVERFLOW_MAX = 5  # 패링 보너스로 초과 가능
const ENERGY_OVERFLOW_DURATION = 2.0  # 2초 후 3으로 감소

var current_energy: float = 0.0
var overflow_timer: float = 0.0

func add_parry_bonus():
    current_energy = min(ENERGY_OVERFLOW_MAX, current_energy + 2.0)
    if current_energy > ENERGY_MAX:
        overflow_timer = ENERGY_OVERFLOW_DURATION
        EnergyUI.show_overflow_glow()  # 에너지 바 황금색 글로우

func _process(delta):
    if overflow_timer > 0:
        overflow_timer -= delta
        if overflow_timer <= 0 and current_energy > ENERGY_MAX:
            current_energy = float(ENERGY_MAX)
            EnergyUI.hide_overflow_glow()
```

**초과 에너지 UI**:
```
기본:    ⚡⚡⚡ (3/3) — 파란색
초과:    ⚡⚡⚡⚡⚡ (5/3) — 황금색 글로우, 2초 카운트다운
```

→ "지금 강력한 카드를 써야 한다!" 라는 시각적 신호.

# ATBIntentSystem.gd
func _on_enemy_atb_updated(enemy: Enemy, atb: float):

## 5. 카드 획득 & 운영에서의 차이 — ATB 버전

### 5.1 패링 특화 덱 vs 회피/방어 특화 덱

ATB에서 방어 스타일은 **전투 리듬 자체**를 다르게 만듦:

```
[패링 특화 플레이어의 ATB 리듬]
에너지 자동충전 (5초) + 패링 성공 (즉시 +2)
= 5초마다 에너지 4~5 보유 가능
= 더 빠른 카드 플레이 → 적 ATB가 올라오기 전에 공격
→ 적극적·공격적 전투 스타일

[방어 특화 플레이어의 ATB 리듬]
에너지 자동충전 + 방어 카드 (즉시 +0.5)
= 안정적이지만 에너지 축적 느림
= 블록을 높이 쌓아 버티며 소량 공격
→ 방어적·지구전 스타일
```

### 5.2 ATB용 고비용 반격 카드

패링 에너지 초과 충전(5) 상황을 노리는 카드 설계:

| 카드명 | 비용 | 효과 | 패링 연계 |
|--------|------|------|---------|
| 꿈의 일격 | 4 | 45 데미지 | 패링 직후 사용 최적 |
| 악몽의 반사 | 3 | 받을 뻔한 피해 × 2 반격 | 패링 직후만 사용 가능 |
| 달빛 폭풍 | 5 | 전체 공격 30 | 패링 2회 연속 시 특화 |
| 역류 | 3 | 다음 에너지 충전 속도 2배 | 오버플로우 상황에 더 빛남 |

### 5.3 자동 에너지 회복과 리액션 에너지의 관계

자동 회복(5초 1 에너지) vs 패링 보너스(즉시 2 에너지):

```
[에너지 타임라인 비교]

패링 없이 자동만:
0초: ⚡⚡⚡ (3)
5초: 소진 후 회복 시작
10초: ⚡⚡⚡ (3) 복구

패링 활용 시:
0초: ⚡⚡⚡ (3)
3초: 패링 성공 → ⚡⚡⚡⚡⚡ (5) 즉시!
→ 카드 4~5 플레이 가능
5초: 자동 충전 루틴 계속
```

→ 패링을 잘 하면 자동 회복 사이클보다 **훨씬 빠른 에너지 흐름** 확보.

## 6. 오토 플레이 연동 — ATB 버전

### 6.1 오토 AI 방어 처리

```gdscript
# ATBAutoAI.gd — 방어 행동 결정

func decide_defense_atb(hand: Array[Card], enemy_atb: float, attack: AttackData) -> String:
    # 에너지가 부족하면 패링 최우선 (에너지 회복 목적)
    if CombatManager.energy < 1 and hand.has_parry_card():
        return "PARRY"  # 에너지 충전 필요 시 적극 패링

    # 강한 공격: 패링 or 방어
    if attack.is_heavy:
        if hand.has_parry_card() and randf() < 0.70:
            return "PARRY"
        return "GUARD"

    # 일반 공격: 회피 선호 (에너지 +1)
    if hand.has_dodge_card():
        return "DODGE"

    # 패링 카드 있으면 패링 (65% 성공률)
    if hand.has_parry_card() and randf() < 0.65:
        return "PARRY"

    return "GUARD"

# 오토 패링 에너지 효율 (수동 100% 대비)
# 오토 패링 성공 시: +2 × 0.65(성공률) = 평균 +1.3
# 수동 패링 성공 시: +2 × 0.9(숙련 성공률) = 평균 +1.8
# → 수동 우위 약 40% 유지
```

## 9. 턴베이스 버전과의 차이점 비교

| 항목 | 턴베이스 버전 | ATB 버전 |
|------|------------|---------|
| 에너지 보너스 시점 | **다음 턴** | **즉시** |
| 전투 리듬 | 순차적, 계획적 | 실시간, 반응적 |
| 패링 에너지 | +2 (다음 턴) | +2 (즉시) |
| 회피 에너지 | +1 (다음 턴) | +1 (즉시) |
| 방어 에너지 | 없음 | +0.5 (즉시) |
| 에너지 초과 | 해당 없음 | 최대 5 (2초) |
| 적 ATB 롤백 | 없음 | 패링 시 50% |
| 카드 드로우 | 패링 시 +1 | 패링 시 +1 |
| 오토 플레이 | 자연스러움 | 자연스러움 |

**핵심 차이**: 턴베이스는 "준비 → 실행" / ATB는 "반응 → 즉발" 흐름.
