# ⚔️ 통합 전투 시스템 설계

**버전**: v1.0 | **날짜**: 2026-03-01 | **상태**: 개발 80% 완료

---

## 📋 개요

이 문서는 Dream Collector의 **ATB & 턴베이스 통합 전투 시스템**의 설계를 정의합니다.

**핵심**: 
- 같은 30장 카드로 두 가지 완전히 다른 게임
- ATB: 실시간 반응 게임
- 턴베이스: 전략 게임

---

## 🎮 시스템 선택 플로우

```
몬스터 전투 (일반)
    ↓
CombatSceneATB 로드
    ↓
CombatManagerATB 실행
    ↓
ATB 게이지 실시간 진행
    ↓
⚡ 실시간 전투

───────────────────────────────────────

보스 전투 (또는 강제 설정)
    ↓
CombatSceneTB 로드
    ↓
CombatManagerTB 실행
    ↓
턴 단계별 진행
    ↓
📋 턴 기반 전투
```

## 🃏 공통 카드 시스템 (30장)

### 카드 구조

```gdscript
Card (Resource)
├─ id: String (고유 ID)
├─ name: String (카드 이름)
├─ cost: int (에너지 비용)
├─ type: String (ATK/DEF/SKILL/POWER/CURSE)
├─ tags: Array[String] (PARRY/DODGE/GUARD/MAJOR_ARCANA)
├─ rarity: String (COMMON/RARE/SPECIAL/LEGENDARY)
├─ damage: int (공격력)
├─ block: int (방어력)
├─ draw: int (드로우 수)
├─ status_effects: Array[Dictionary] (상태이상)
└─ description: String (설명)
```

### 카드 분류

**공격 (ATK) — 10장**
```
비용 1: 검의에이스, 번개
비용 2: 이중베기, 마법사, 탑, 악마, 별
비용 3: 태양, 황제
비용 4: 세계

특징: 낮은 비용 카드가 많아 초반 사용 용이
```

**방어 (DEF) — 8장**
```
비용 1: 철벽, 정의, 절제
비용 2: 방패의왕, 교황, 달
비용 3: 여황제
비용 2+: 은둔자

특징: 블록값 다양, 일부 부가 효과 (힐, 드로우)
```

**패링 (PARRY) — 5장**
```
모두 0비용 (또는 1비용)
특징: 리소스 부담 적음, 리액션 전용

꿈의쳐내기 → 에너지+2 + 드로우1
반사의순간 → 에너지+2 + 반격30%
각성의쳐내기 → 에너지+3 (윈도우 좁음)
달빛반격 → 에너지+1 + 즉시 8공격
완벽한방어 → 패링/회피 겸용
```

**회피 (DODGE) — 5장**
```
모두 0비용 (또는 1비용)
특징: 리소스 부담 적음, 리액션 전용

꿈의스텝 → 에너지+1
잔상 → 에너지+1 + 버프이전
황혼의도약 → 에너지+1 + 다음공격+3
연막 → 에너지+1 + 적약화
반보앞으로 → 감소 블록 + 에너지+1
```

**스킬 (SKILL) — 2장**
```
바보 (0비용) → 드로우1 + 에너지+1
달의환영 (2비용) → 드로우3
```

## ⚡ 에너지 시스템

### ATB 에너지 메커닉

```
시간 흐름:
  0초: 에너지 0
  5초: 에너지 1 (자동)
  10초: 에너지 2 (자동)
  ...
  매 5초: +1 (무한)

방어 보상 (즉시):
  패링: +2
  회피: +1
  방어: +0.5

상한: 5 (3초간 유지 후 강제 복구)

플레이어 관점:
  - 기다리면 에너지 늘어남
  - 방어 잘하면 더 빨리 공격 가능
  - = 에너지 자원 획득 시스템
```

### 턴베이스 에너지 메커닉

```
턴 시작:
  기본: 3
  + 이전 턴 방어 보너스 (1~2)

최대 오버플로우: 5

방어 보상 (다음 턴):
  패링: +2
  회피: +1
  방어: 보너스 없음 (블록만)

플레이어 관점:
  - 이번 턴 받은 카드로 계획 수립
  - 방어 잘하면 다음 턴이 강해짐
  - = 턴별 전략 수립
```

## 🎯 리액션 시스템

### 공통 리액션 프레임워크

```
리액션 윈도우:
  ├─ 패링: 0.8초 (Story) / 0.5초 (Hard)
  └─ 회피: 1.8초 (Story) / 1.2초 (Hard)

판정 로직:
  1. 패링 윈도우 내? → PARRY
  2. 회피 윈도우 내? → DODGE
  3. 방어 카드? → GUARD
  4. 시간 초과? → NONE

특수:
  UNBLOCKABLE 공격: 패링 불가 (회피만)
```

### ATB 리액션 플로우

```
적 ATB 100% 도달
    ↓
리액션 윈도우 오픈 (0.8초)
    ↓
플레이어 패링/회피 카드 탭?
    ├─ YES (0.8초 내): PARRY
    │   └─ 에너지 +2, 적 ATB 50% 롤백
    │
    ├─ YES (0.8~1.8초): DODGE
    │   └─ 에너지 +1
    │
    ├─ YES (방어): GUARD
    │   └─ 에너지 +0.5
    │
    └─ NO (1.8초 후): NONE
        └─ 피해 받음
```

### 턴베이스 리액션 플로우

```
적 턴 시작
    ↓
의도 공개 (2~3행동 미리보기)
    ↓
플레이어 대기 (리액션 준비)
    ↓
적 행동 실행
    ↓
리액션 윈도우 오픈 (0.8초)
    ↓
플레이어 카드 탭
    ├─ PARRY: 다음 턴 에너지 +2
    ├─ DODGE: 다음 턴 에너지 +1
    ├─ GUARD: 블록만
    └─ NONE: 피해
```

## 🎪 콤보 시스템

### 콤보 감지 로직

```
최근 카드 히스토리 추적 (최대 5장)
    ↓
카드 플레이할 때마다 체크:

1. 연타 (ATK 3장)
   if [마지막 3장 모두 ATK]:
     damage *= 1.75  // +75%

2. 패링 반격
   if [마지막 2장: PARRY → ATK]:
     damage *= 1.30  // +30%

3. 약점 폭로
   if [마지막 1장에 VULNERABLE상태 + 이번 ATK]:
     damage *= 1.50  // +50%

4. 완벽한 방어
   if [마지막 2장 모두 DEF]:
     블록만 누적 (보너스 없음)
```

### 콤보 UI

```
플레이어 화면:
├─ 콤보 힌트 미리보기
│  ("공격 1장 더 → 연타 콤보!")
│
├─ 콤보 발동 시 (COMBO! 3x)
│  VFX: 파티클 폭발
│  SFX: combo_hit
│
└─ 콤보 카운터
   (이번 턴 누적 콤보 수)
```

## 🌊 상태이상 시스템

### 7가지 상태이상

```
POISON (중독)
├─ 매 턴 HP 감소 (스택 가능)
├─ 시각: 자줏빛 오라
└─ 사용 카드: 악마 (중독+3)

VULNERABLE (취약)
├─ 받는 데미지 +50%
├─ 시각: 빨간 X 마크
└─ 사용 능력: 꿈 조각 (NIGHTMARE)

WEAK (약화)
├─ 주는 데미지 -25%
├─ 시각: 회색 다운 화살표
└─ 기타 카드 효과

STRENGTH (힘)
├─ 공격력 증가 (플레이어용)
├─ 시각: 금색 업 화살표
└─ 사용 카드: 황제 (힘+2)

DEXTERITY (민첩)
├─ 블록 획득량 증가 (방어 강화)
├─ 시각: 파란색 스파크
└─ 사용 카드: 달 (민첩+1)

BURNING (화상)
├─ 매 턴 HP 감소 (POISON과 유사)
├─ 시각: 주황색 불꽃
└─ 상태이상 체크 필요

ENTANGLED (묶음)
├─ 이번 턴 공격 불가
├─ 시각: 갈색 줄
└─ 무력화 상태
```

## 🎪 ATB 특수 시스템

### 의도 표시 (ATB 기반)

```
ATB 게이지에 따른 강도 변화:

0~60%: 일반
├─ 아이콘 작음
├─ 컬러: 회색
└─ 피드백: 없음

60~80%: 강조
├─ 아이콘 중간
├─ 컬러: 노란색
└─ 피드백: 경고음 약함

80~95%: 경고
├─ 아이콘 크게 흔들림
├─ 컬러: 주황색
├─ 피드백: 경고음 강함
└─ 시각: 테두리 강조

95%+: 위험
├─ 아이콘 점프, "⚡위험!" 텍스트
├─ 컬러: 빨간색
├─ 피드백: 진동 + 사이렌음
└─ 시각: 화면 깜빡임
```

### 집중 모드

```
조건: 에너지 1 소모 (플레이어 선택)

효과:
├─ 시간 배율: 0.3배 (느려짐)
├─ 지속: 3초
└─ 복귀: 자동 (시간 종료)

시각:
├─ 화면 왜곡 (시간 편향 표현)
├─ 슬로우 모션 샤우팅
└─ UI: "집중 모드" 텍스트

의도: 어려운 공격 회피의 기회 제공
```

### 위기 모드

```
조건: HP 30% 이하 (자동 발동)

효과:
├─ 시간 배율: 0.5배 (느려짐)
├─ 지속: 10초
└─ 복귀: 자동 (시간 종료 또는 HP 회복)

시각:
├─ 화면 가장자리 빨간 테두리
├─ 맥박 효과 (주기적)
└─ UI: "위기!" 텍스트 + 심장 사운드

의도: 죽음 직전 역전의 기회
      게임 난이도 관대하게 조정
```

## 📋 턴베이스 특수 시스템

### 손패 및 드로우

```
드로우: 턴당 5장
최대: 10장
초과: 드로우 불가 (다음 턴까지 대기)

버림더미:
├─ 사용한 카드
├─ 턴 종료 시 미사용 카드
└─ 모든 카드 소진 후 자동 셔플

룰:
├─ 턴 시작: 드로우
├─ 턴 진행: 카드 사용
└─ 턴 종료: 미사용 카드 버림
```

### 타로 에너지 (Tarot Energy)

```
최대: 3

충전:
├─ 메이저 아르카나 카드 사용 시 +1
└─ 카드 예시: (MAJOR_ARCANA 태그)

사용:
├─ 특정 강력 카드의 비용
└─ 예: "달의 환영" = 타로 2 소모

의도: 강력한 카드를 더 전략적으로
      덱 구성에 따라 플레이 스타일 변화
```

### 꿈 조각 (Dream Shard)

```
최대: 5

획득:
├─ 패링 성공 시 +1
├─ 특정 콤보 완성 시 +1
└─ 연속 방어 성공 시 +1

사용 능력:
├─ QUICK_DRAW (1 소모): 카드 드로우
├─ ENERGY_BURST (2 소모): 에너지+1
├─ DREAM_HEAL (3 소모): HP +8
└─ NIGHTMARE (5 소모): 모든 적 약화

의도: 방어 우수 플레이어가 강해짐
      위기 상황에서 역전 가능
```

### 덱 패시브

```
덱 구성에 따라 자동 적용:

달의 기사 (DEF≥5):
└─ 턴 시작 블록+3

검의 달인 (ATK≥7):
└─ 첫 공격 데미지+2

타로 학자 (MAJOR_ARCANA≥3):
└─ 전투 시작 타로+1

달빛 반격사 (PARRY≥4):
└─ 패링 보너스+1

의도: 덱 빌드의 중요성 강조
      플레이 스타일 다양화
```

## 🤖 AI 시스템 (공통)

### 3가지 모드

```
MANUAL (수동)
├─ AI 없음
└─ 플레이어만 조작

SEMI (세미 오토)
├─ AI가 추천 카드 강조
├─ 플레이어가 최종 결정
└─ 도움말 역할

FULL (풀 오토)
├─ AI가 자동으로 플레이
├─ 플레이어는 감상
└─ 영상미감상용
```

### AI 우선순위

```
ATB:
1. 적 ATB 80%+ → 방어 준비
2. 플레이어 HP 30% 이하 → 방어/힐
3. 적 상태 약화 없음 → 디버프
4. 콤보 가능 → 콤보 완성
5. 비용/효율 최고 → 일반 공격

턴베이스:
1. 플레이어 HP 30% 이하 → 적극 방어
2. 높은 피해 공격 예상 → 방어 준비
3. 이전 턴 회피 보너스 있으면 → 공격
4. 상태이상 카드 → 이점 공격
5. 비용/효율 최고 → 일반 공격
```

## 📊 성능 고려사항

### 메모리 최적화
```
카드: Resource로 한 번만 로드
몬스터: 오브젝트 풀링 (재사용)
이펙트: 풀 기반 렌더링
```

### 프레임 최적화
```
ATB: 프레임 독립적 (delta time 사용)
턴베이스: 턴 진행 동안 프레임 여유 있음
의도 표시: 업데이트 최소화
```
