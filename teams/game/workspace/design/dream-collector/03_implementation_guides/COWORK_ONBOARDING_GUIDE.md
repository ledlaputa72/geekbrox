# 🎮 Dream Collector — Cowork 온보딩 가이드

**목적:** Cowork가 Dream Collector의 전체 게임 설계를 빠르게 이해하고 개선안을 도출하기 위한 학습 가이드  
**대상:** 신규 기획자/PM/협력팀  
**학습 시간:** 2-3시간 (빠른 스캔) / 6-8시간 (상세 학습)  
**마지막 업데이트:** 2026-03-06

---

## 📚 학습 로드맵 (2시간 집중 학습)

### 1️⃣ **전체 게임 이해 (15분)**
📖 **읽기:** [01_vision 폴더의 게임 비전]
- Dream Collector 핵심 개념
- "The Mnemonic Weaver" 스토리
- Roguelike + Deckbuilding + Idle 장르 혼합

**알아야 할 것:**
- 프로토타입(Nox)의 목표: 잃어버린 꿈 수집
- 핵심 루프: 가샤 선택 → 카드 조합 → 전투 → 꿈 수집

---

### 2️⃣ **캐릭터 시스템 이해 (30분)**
📖 **읽기:** [CHARACTER_SYSTEM_DETAILED.md] ← NEW
- 캐릭터 기본 속성 (Lv 1-100)
- 6가지 핵심 스탯 (STR, DEF, VIT, DEX, INT, LUK)
- 특성(Traits) 시스템 (건강함, 민첩함, 지혜 등)
- 스탯과 특성의 상호작용

**알아야 할 것:**
```
캐릭터 = 기본 스탯 (Lv 기반) + 특성 (선택적) + 장비 보너스
```

---

### 3️⃣ **카드 시스템 이해 (30분)**
📖 **읽기:** [CARD_SYSTEM_SUMMARY.md] ← NEW
- 200개 카드의 4가지 타입 (ATTACK/SKILL/POWER/CURSE)
- 카드 효과 체계 (데미지, 버프, 디버프)
- 카드 비용(Mana) & 쿨다운
- 덱 빌딩 전략

**알아야 할 것:**
```
카드 선택 → 덱 빌드 → 전투에서 실행 → 카드 조합이 승패 결정
```

---

### 4️⃣ **아이템 시스템 이해 (20분)**
📖 **읽기:** [ITEM_SYSTEM_SUMMARY.md] ← NEW
- 90개 아이템 (무기 20, 방어구 20, 반지 25, 목걸이 25)
- 5가지 등급 (Common ~ Legendary)
- 각 아이템의 특성과 보너스

**알아야 할 것:**
```
장비 = 스탯 강화 + 특수 능력
└─ 무기: ATK, 스킬 강화
└─ 방어구: DEF, HP 증가
└─ 반지: 캐릭터 스탯 강화
└─ 목걸이: 카드 효율 증가
```

---

### 5️⃣ **전투 시스템 이해 (25분)**
📖 **읽기:** [COMBAT_INTEGRATION_GUIDE.md] ← NEW
- ATB (Active Time Battle) 기본 메커닉
- 카드 선택 → 액션 실행 → 효과 적용 흐름
- 장비와 카드의 시너지
- 전투 밸런싱 원칙

**알아야 할 것:**
```
전투 = ATB 속도 관리 + 카드 선택 + 장비 시너지
└─ 카드 타입별 전략 (ATTACK 빠름, POWER 강함)
└─ 장비 조합으로 시너지 극대화
```

---

### 6️⃣ **경제 시스템 이해 (15분)**
📖 **읽기:** [03_implementation_guides/operations/GAME_ECONOMY_MANAGEMENT.md]
- 골드 & 보석 획득 경로
- 월간 보상 분배 전략
- 인플레이션 관리 (5-10% 목표)

**알아야 할 것:**
```
경제 = 무료 게임의 장기 플레이 지원 + 과금자 가치 제공
```

## 💡 학습 팁

### ✅ **효과적인 학습 방법**
1. **텍스트를 읽으면서 메모 작성** (핵심 포인트 정리)
2. **각 파일마다 Q&A 작성** ("왜 이렇게 설계했는가?")
3. **실제 게임을 생각하며 학습** (플레이어 관점)
4. **데이터를 비교 분석** (아이템 등급별, 카드 타입별)

### ⚠️ **주의할 점**
- 문서는 "설계" 기준 (실제 게임과 다를 수 있음)
- 데이터는 밸런싱 대상 (변경될 수 있음)
- 시스템은 상호 의존적 (한 시스템 변경 = 다른 시스템 영향)

### 🎯 **학습 후 기대 효과**
- ✅ 게임 시스템 전체 이해
- ✅ 각 시스템의 상호작용 파악
- ✅ 부족한 부분 식별
- ✅ 개선안 도출 가능
