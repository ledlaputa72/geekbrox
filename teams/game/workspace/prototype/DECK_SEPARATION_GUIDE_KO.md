**목적:** 플레이어 카드와 몬스터 카드의 명확한 분리

## 🎴 개요: 두 개의 구별된 카드 시스템

### 플레이어 카드 (총 61장)
- 플레이어가 자신의 턴에 사용하는 카드
- 플레이어가 관리 (GM 도움 포함)
- 구성: 드로우 파일, 핸드, 버린 카드 더미

### 몬스터 카드 (총 34장)
- GM이 적을 나타내기 위해 사용하는 카드
- GM만 관리
- 구성: 몬스터 덱, 활성 몬스터, 격파 더미

**핵심 규칙:** 이 두 덱은 절대 섞이지 않습니다. 물리적으로 분리하세요.

## 🃏 플레이어 카드 시스템 (61장)

### 타입별 카드 분포

| 타입 | 수량 | 색상 코드 | 목적 |
|------|------|-----------|------|
| 공격 | 18 | 빨간 테두리 | 몬스터에게 피해 |
| 방어 | 12 | 파란 테두리 | 몬스터 피해 막기 |
| 수집 | 16 | 금색 테두리 | 레버리 생성 |
| 시너지 | 15 | 보라 테두리 | 콤보 효과 |

### 플레이어 덱 구조 (게임 중)

```

```

### 캐릭터별 시작 덱

**세레니티 (균형형):**
- 3× 기억 파편 (수집)
- 2× 기본 공격 (공격)
- 1× 꿈의 방패 (방어)
- 2× 레버리 폭발 (수집)

**비고르 (공격형):**
- 4× 기본 공격 (공격)
- 2× 퀵 슬래시 (공격, 0 비용)
- 1× 꿈의 방패 (방어)
- 1× 레버리 폭발 (수집)

**큐레이터 (경제형):**
- 4× 기억 파편 (수집)
- 2× 레버리 폭발 (수집)
- 1× 기본 공격 (공격)
- 1× 꿈의 방패 (방어)

*(모든 12개 시작 덱은 CHARACTER_CARDS_KO.md 참조)*

### 플레이어 카드 풀 (보상 풀)

**남은 53장** = 런 중 추가를 위한 보상 풀

**GM이 별도 더미에 보관:**
- 플레이어가 전투 승리 → 랜덤 3장 드로우
- 플레이어에게 보여주기
- 플레이어가 1장 선택해 덱에 추가
- 나머지 2장은 풀로 되돌리기 (섞기)

**정리 팁:**
더 쉬운 드로우를 위해 희귀도별로 보상 풀 정렬:
- 일반 (20장) - 녹색 슬리브
- 언커먼 (20장) - 파란 슬리브
- 레어 (10장) - 보라 슬리브
- 에픽 (3장) - 금색 슬리브

## 👾 몬스터 카드 시스템 (34장)

### 등급별 카드 분포

| 등급 | 수량 | 색상 코드 | 목적 |
|------|------|-----------|------|
| 기본 | 14 | 흰색 테두리 | 일반 인카운터 |
| 엘리트 | 10 | 보라 테두리 | 선택적 어려운 전투 |
| 보스 | 10 | 빨간 테두리 | 최종 인카운터 |
| **합계** | **34** | | |

### 몬스터 덱 구조 (게임 중)

```
┌─────────────────────────────────────┐
│ GM 영역 (몬스터)                    │
├─────────────────────────────────────┤
│                                     │
│  [기본 몬스터]  [활성]  [격파]      │
│   덱 (14)      몬스터   더미        │
│   (뒷면)       (앞면)   (뒷면)      │
│                                     │
│  [보스 덱]                          │
│   (10장, 1개 사전 선택)             │
│                                     │
└─────────────────────────────────────┘
```

### 몬스터 덱 준비 (GM 설정)

**게임 시작 전:**

1. **기본 몬스터 덱 (14장):**
   - 모든 14장 기본 몬스터 카드 섞기
   - GM 영역에 뒷면으로 배치
   - 이것이 "인카운터 덱"

2. **보스 덱 (10장):**
   - 섞지 말기
   - 난이도 기반으로 노드 10용 보스 1개 사전 선택
   - 선택한 보스를 옆에 두기 (뒷면)
   - 나머지 9개 보스는 박스에 다시 넣기

3. **엘리트 몬스터 덱 (10장) - 선택:**
   - 엘리트 인카운터 사용 시 (5% 확률)
   - 기본 덱과 분리 유지
   - 특수 이벤트 트리거 시 드로우

## 🎲 전투 흐름: 카드 상호작용

### 설정 페이즈 (전투 전)

**GM:**
1. 기본 몬스터 덱에서 몬스터 카드 1장 드로우
2. 활성 몬스터 구역에 앞면으로 배치
3. 공지: "당신은 [몬스터 이름]과 마주합니다!"
4. 스탯 읽기: HP, 공격, 특수 능력

**플레이어:**
5. 설정 필요 없음 (이미 5장 핸드 보유)

### 전투 페이즈 (매 턴)

```
┌──────────────────────────────────────┐
│ 턴 구조                              │
├──────────────────────────────────────┤
│ 1. 플레이어 턴                       │
│    - 드로우 파일에서 카드 1장 드로우 │
│    - 핸드에서 카드 플레이            │
│    - 각 카드 → 버린 카드 더미        │
│    - 활성 몬스터에게 피해            │
├──────────────────────────────────────┤
│ 2. 몬스터 턴 (GM 제어)               │
│    - 몬스터 패턴 읽기                │
│    - 공격 값 공지                    │
│    - 플레이어가 방어 플레이 가능     │
│    - 플레이어에게 피해 계산          │
├──────────────────────────────────────┤
│ 3. 승리 확인                         │
│    - 몬스터 HP = 0? → 플레이어 승리  │
│    - 플레이어 HP = 0? → 플레이어 패배│
│    - 둘 다 아님? → 다음 턴           │
└──────────────────────────────────────┘
```

## 🚫 피해야 할 일반적인 실수

### 실수 #1: 플레이어와 몬스터 카드 섞기

**문제:** 플레이어가 몬스터 카드를 플레이하려고 함.

**수정:** 덱을 테이블 반대편에 유지. 다른 카드 슬리브 색상 사용.

---

### 실수 #2: 플레이어 보상 풀에 몬스터 있음

**문제:** GM이 실수로 보상으로 몬스터 카드 드로우.

**수정:** 게임 전에 사전 분리. 몬스터 카드는 절대 플레이어에게 가지 않음.

---

### 실수 #3: 격파한 몬스터 재섞기

**문제:** 같은 몬스터가 두 번 등장.

**수정:** 격파한 몬스터는 격파 더미로 (게임에서 제외). 재섞지 말 것.

---

### 실수 #4: 플레이어가 몬스터 덱에서 드로우

**문제:** 플레이어가 잘못된 더미에서 드로우.

**수정:** 덱을 명확히 라벨링. 플레이어는 자신의 드로우 파일에서만 드로우.

---

### 실수 #5: 보상 카드를 덱에 추가하는 것 잊기

**문제:** 플레이어가 보상 카드를 선택했지만 추가하는 것을 잊음.

**수정:** GM이 카드를 플레이어에게 물리적으로 건네며 "지금 버린 카드 더미에 추가하세요."
