# 크레딧 Cascade 빠른 참조 카드

**언제든 참고하세요!** 📋

---

## 🎯 3단계 전략

```
┌─────────────────────────────────────┐
│ Level 1: Cursor (무료 로컬 모델)     │
│ ✅ 최우선 사용                       │
│ ✅ 무제한                            │
│ ✅ 가장 빠름                         │
└──────────────┬──────────────────────┘
               │ 크레딧 소진?
               ↓
┌─────────────────────────────────────┐
│ Level 2: Claude Code (무료 API)     │
│ ⚠️ 일일 한도 ~500 메시지             │
│ ⚠️ 자정 UTC 리셋 (오전 9시 PST)      │
└──────────────┬──────────────────────┘
               │ 크레딧 소진?
               ↓
┌─────────────────────────────────────┐
│ Level 3: Atlas (유료 API)           │
│ 🚨 최후 수단                         │
│ 🎯 전체 정리/관리/복잡 작업          │
│ 💰 ~$0.01-0.05/메시지               │
└──────────────┬──────────────────────┘
               │ 리셋 대기
               ↓
          Level 1 복귀
```

---

## ✅ 언제 뭘 쓸까?

### 일반 작업 (80%) → Cursor
- 버그 수정
- 코드 수정
- 작은 기능
- 리팩터링

### 복잡한 작업 (15%) → Claude Code
- 대화형 디버깅
- 단계별 구현
- 에러 분석

### 전략적 작업 (5%) → Atlas
- 아키텍처 설계
- 전체 정리
- 문서화
- Git/Notion 관리

---

## 🔄 전환 방법

### Cursor → Claude Code
```bash
# 증상: "Rate limit" 메시지

# 절차:
git add . && git commit -m "WIP: [작업 내용]"
# CHANGELOG 업데이트
code .  # VSCode 열기
# @PROJECT_CONTEXT.md
# 이어서 작업
```

### Claude Code → Atlas
```bash
# 증상: "Rate limit exceeded"

# 절차:
git add . && git commit -m "WIP: [작업 내용]"
# CHANGELOG 업데이트
# 텔레그램 → Atlas
```

**템플릿**:
```
🚨 크레딧 소진

Cursor: ❌
Claude Code: ❌

요청: [정리/공유/복잡 작업]
```

## 💡 절약 팁

### Cursor
- Settings → Local Model 우선
- 자유롭게 사용 (무제한!)

### Claude Code
- 짧고 명확한 요청
- @-mention 활용
- 불필요한 대화 최소화

### Atlas
- 여러 질문 한 번에
- 주 1-2회 정기 리뷰
- 일상 작업은 Level 1-2
