# Dream Collector 전투 시스템 분석 보고서

**작성일:** 2026-03-03  
**목적:** 개발된 게임(전투) 시스템의 구조·기능·연동 현황 파악 및 정리

---

## 1. 개요

Dream Collector 전투는 **이중 모드**로 구성되어 있다.

| 구분 | 일반 전투 | 보스 전투 |
|------|-----------|-----------|
| **모드** | ATB (Active Time Battle) | 턴베이스 (Turn-Based) |
| **진입** | 맵 노드 `combat` | 맵 노드 `boss` |
| **특징** | 실시간 ATB 게이지 + 카드 플레이 | 플레이어 턴 → 적 턴 순차, 리액션 윈도우 |
| **씬** | `CombatSceneATB.tscn` | `CombatSceneTB.tscn` |
| **매니저** | `CombatManagerATB` | `CombatManagerTB` |

- **엔진:** Godot 4.x / GDScript  
- **해상도:** 390×844 (Portrait)  
- **설정:** `SettingsManager`에서 스토리/하드 모드, 전투 속도, F1/F2 강제 모드 지원

## 2. 아키텍처

### 2.1 폴더 구조

```
scripts/combat/
├── shared/                    # 두 모드 공통
│   ├── Card.gd                 # 카드 리소스
│   ├── CardDatabase.gd        # 카드 30종 (Autoload)
│   ├── Monster.gd             # 몬스터 엔티티
│   ├── StatusEffectSystem.gd  # 상태이상
│   └── BattleDiary.gd         # 전투 로그/통계
├── atb/                        # ATB 전투
│   ├── CombatManagerATB.gd
│   ├── ATBEnergySystem.gd
│   ├── ATBReactionManager.gd
│   ├── ATBIntentSystem.gd
│   ├── ATBComboSystem.gd
│   ├── ATBAutoAI.gd
│   ├── ATBFocusMode.gd
│   └── ATBCrisisMode.gd
└── turnbased/                  # 턴베이스 전투
    ├── CombatManagerTB.gd
    ├── TurnBasedEnergySystem.gd
    ├── TurnBasedReactionManager.gd
    ├── TurnBasedHandSystem.gd
    ├── TurnBasedIntentSystem.gd
    ├── TarotEnergySystem.gd
    ├── DreamShardSystem.gd
    ├── DeckPassiveCalculator.gd
    └── TurnBasedAutoAI.gd
```

### 2.2 전투 진입 흐름

1. **인런 화면:** `InRun_v4.gd` — 맵 노드 도달 시 `_on_node_reached()`  
2. **노드 타입:** `combat` → `_handle_combat_event()` → `switch_to_combat(false)` (ATB)  
3. **보스:** `boss` → `_handle_boss_event()` → `switch_to_combat(true)` (턴베이스)  
4. **switch_to_combat(is_boss):**  
   - `CombatBottomUI` 로드 및 `connect_combat_manager(manager)`  
   - 몬스터 스폰 → `_start_atb_combat()` 또는 `_start_tb_combat()`  
   - `CombatManagerATB.start_combat()` / `CombatManagerTB.start_combat()` 호출  
5. **전투 종료:** 매니저가 `_check_battle_end()` 후 `combat_ended("WIN"|"LOSE")` 발신 → InRun_v4가 보상/패배 처리 후 탐험으로 복귀

## 3. 공통 시스템

### 3.1 카드 (Card.gd)

- **Resource** 기반, `CardDatabase`에서 로드  
- **메인 타입:** ATK, SKILL, POWER, CURSE  
- **서브/태그:** GUARD, PARRY, DODGE, MAJOR_ARCANA, AOE 등  
- **주요 필드:** cost, damage, block, draw, status_effects, auto_dodge_success_rate  
- **메서드:** has_tag(), get_mobile_description(), duplicate_card(), dmg_per_energy()

### 3.2 카드 데이터베이스 (CardDatabase.gd, Autoload)

- **약 30종** 카드 정의 (공격·방어·패링·회피·스킬 등)  
- `get_starter_deck()`: 스타터 덱 반환  
- `get_by_id()`, `get_by_sub_category()` 등으로 조회

### 3.3 몬스터 (Monster.gd)

- **Node** 기반, 런타임에 `current_hp`, `atb`, `block`, `status_effects`  
- **행동 패턴:** action_patterns (NORMAL, HEAVY, UNBLOCKABLE, DEFEND, BUFF 등)  
- **API:** is_alive(), take_damage(), get_next_action(), get_action_queue(), make_attack_data()

### 3.4 상태이상 (StatusEffectSystem.gd)

- **7종+:** POISON, VULNERABLE, WEAK, STRENGTH, DEXTERITY, BURNING, ENTANGLED, HEAL  
- apply(), tick_turn(), get_damage_multiplier(), get_outgoing_multiplier()  
- 취약 +50% 피해, 약화 -25% 피해 등 적용

## 4. ATB 전투 시스템

### 4.1 핵심 루프

- **CombatManagerATB._process(delta):**  
  - ATB 게이지 충전 (`_update_atb`)  
  - 에너지 타이머 (5초마다 +1, 최대 3·오버플로우 시 3초간 최대 5)  
  - 위기 모드 체크 (HP ≤ 30% 시 슬로우 등)  
- **적 ATB 100% 시:**  
  - `reaction_mgr.open_reaction_window(attack)`  
  - `await reaction_resolved`  
  - `_apply_attack_result()` (PARRY/DODGE/GUARD/NONE)

### 4.2 리액션 (ATBReactionManager)

- **윈도우:** 녹(방어 전체) → 노(회피) → 빨(패링만), Story/Hard 별 구간 시간 상수  
- **open_reaction_window(attack)** → reaction_state = "OPEN", 시그널 발신  
- **on_player_tap_card(card):**  
  - UNBLOCKABLE이면 PARRY 불가, DODGE만 유효  
  - phase에 따라 PARRY(빨)/DODGE(노·빨)/GUARD(녹·노·빨) 성공·실패 판정  
- **시그널:** reaction_window_opened/closed, parry_success, dodge_success, guard_success, reaction_attempt_failed

### 4.3 에너지 (ATBEnergySystem)

- 기본 3, 5초마다 +1  
- 방어 성공 시: 패링 +2, 회피 +1, 가드 +0.5 (오버플로우 허용 후 3초 뒤 3으로 감소)

### 4.4 콤보 (ATBComboSystem)

- 연타(ATK 3연속), 완벽한 방어(DEF 2연속), 패링 반격, 약점 폭로 등 4종  
- get_next_combo_hint()로 UI 힌트 제공

### 4.5 부가 시스템

- **ATBFocusMode:** 에너지 1 소모 → 속도 0.3배 3초  
- **ATBCrisisMode:** HP ≤ 30% 시 1회, 속도 0.5배 10초  
- **ATBAutoAI:** MANUAL/SEMI/FULL, 적 ATB·HP·디버프·코스트 효율 기반 카드 선택

## 5. 턴베이스 전투 시스템

### 5.1 턴 흐름

- **플레이어 턴:**  
  - 에너지 3 + 패링/회피 보너스, 손패 드로우, 의도 표시  
  - 카드 플레이 또는 턴 종료  
- **적 턴:**  
  - 각 적 순서대로 행동  
  - 행동 전 **리액션 윈도우** (패링/회피/가드)  
  - DoT 등 상태이상 틱

### 5.2 에너지·손패 (TurnBasedEnergySystem, TurnBasedHandSystem)

- BASE_ENERGY 3, 패링 성공 +2, 회피 +1 (다음 턴 보너스)  
- HAND_MAX 10, 턴 종료 시 discard_remaining()

### 5.3 리액션 (TurnBasedReactionManager)

- ATB와 유사한 phase(녹/노/빨) 윈도우  
- open_window(attack), on_player_card_tapped(card)  
- reaction_window_opened/closed, reaction_resolved 시그널

### 5.4 턴베이스 전용

- **TarotEnergySystem:** 메이저 아르카나 사용 시 타로 에너지 +1 (최대 3), 타로 스킬 소비  
- **DreamShardSystem:** 패링 성공 시 꿈 조각 +1 (최대 5), 스킬 소모  
- **DeckPassiveCalculator:** 덱 구성을 보고 패시브(달의 기사, 검의 달인 등) 적용  
- **TurnBasedAutoAI:** decide_defense(가드 우선, UNBLOCKABLE 시 회피만), decide_attack_cards(HP 비율·효율 기반)

## 6. UI 연동

### 6.1 CombatBottomUI

- **역할:** 카드 손패, 에너지, 덱/버림, 전투 로그, 패스/오토/스피드, **리액션 버튼**  
- **connect_combat_manager(manager):**  
  - hand_updated, energy_updated, combat_ended 등 시그널 연결  
  - reaction_mgr이 있으면 reaction_window_opened/closed, reaction_attempt_failed 연결  
- **리액션 버튼:** 리액션 윈도우가 열려 있을 때만 패링/회피/방어 카드 사용 가능하도록 활성화  
- **카드 탭/드래그:** player_play_card(card), 타겟 선택 시 damage_dealt 등 연동

### 6.2 InRun_v4

- TopArea: 히어로·몬스터 캐릭터 노드  
- BottomArea: ExplorationBottomUI / CombatBottomUI 등 동적 전환  
- 전투 종료 시 보상 모달 또는 패배 화면 후 `switch_to_exploration()`  
- F1(ATB 강제), F2(턴베이스 강제), F3(해제), 2/B 키 테스트 진입

---

## 7. 전투 종료·승패

- **CombatManagerATB / CombatManagerTB:**  
  - `_check_battle_end()`: 플레이어 HP ≤ 0 → LOSE, 모든 적 사망 → WIN  
  - `_end_combat(result)` → `emit_signal("combat_ended", result)`  
- **InRun_v4:**  
  - `_on_new_combat_ended(result)` → 승리 시 보상 적용·모달, 패배 시 패배 화면  
  - 이후 탐험으로 전환
