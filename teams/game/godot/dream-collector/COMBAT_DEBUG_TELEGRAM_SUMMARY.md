# Dream Collector 전투 시스템 디버깅 완료 보고

**날짜**: 2026-03-01

---

## 핵심 요약

전투 시스템 전체 코드 리뷰 및 디버깅 완료. **총 23건의 버그** 발견 및 수정.

### 해결된 주요 이슈

**1. 전투 진입 시 크래시 (CRITICAL)**
- 원인: Monster의 `action_patterns`에 타입 불일치 배열 대입 → Godot 4.x 런타임 에러
- 수정: 타입 안전한 `Array[Dictionary]` + `.assign()` 사용

**2. 전투 UI에 카드가 표시되지 않는 문제 (CRITICAL)**
- 원인: CombatBottomUI가 기존 CombatManager autoload에만 연결되어 있고, 새로 개발된 CombatManagerATB/TB와 연결되지 않음
- 수정: `connect_combat_manager()` 브릿지 메서드 구축. Card Resource → Dictionary 변환, 시그널 연결, 카드 사용 라우팅 전체 구현

**3. 전투 로직 버그 다수 (HIGH: 12건)**
- ATK 카드 전부 AoE로 동작하던 문제 → 단일/AoE 분기 추가
- 콤보 시스템 등록 타이밍 오류 → 순서 수정
- 위기/집중 모드 속도 충돌 → 크로스체크 로직 추가
- 리액션 윈도우 이중 타이머 → 단일 타이머로 통합
- 상태이상(VULNERABLE/WEAK) 미감소 → 감소 로직 추가
- 턴베이스: await 후 combat_active 가드 누락 → 가드 삽입
- 기타 4건
