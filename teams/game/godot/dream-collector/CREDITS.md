# 크레딧 관리 & Cascade 전략

**작성**: Atlas (2026-02-25)  
**목적**: 유료 크레딧을 최대한 효율적으로 사용하기 위한 3단계 전략

---

## 🎯 Cascade 전략 개요

```
Level 1: Cursor IDE (무료 로컬 모델)
   ↓ 크레딧 소진 시
Level 2: Claude Code (무료 API 티어)
   ↓ 크레딧 소진 시
Level 3: Atlas (유료 Claude API)
   ↓ 크레딧 리셋 대기
Level 1로 복귀
```

**목표**: 유료 API 사용 최소화, 무료 리소스 최대 활용

## 🔄 Level 1: Cursor IDE (최우선)

### 사용 시점
- ✅ **기본 선택**: 모든 작업 첫 번째로 시도
- ✅ 로컬 모델 무제한 사용
- ✅ 가장 빠르고 비용 없음

### 크레딧 정보
- **무료 사용**: 로컬 모델 (GPT-4o-mini, 기타)
- **제한**: 없음 (로컬 실행)
- **속도**: 매우 빠름

### 사용법
```bash
cd ~/Projects/geekbrox/teams/game/godot/dream-collector
cursor .

# Cmd+I (Composer)
"작업 내용"
```

### 최적화 팁
- **Settings → Models → Local**: 로컬 모델 우선 선택
- **큰 작업도 OK**: 무제한이므로 자유롭게
- **Composer 활용**: 멀티파일 수정 강력

### 크레딧 소진 시
```
Cursor에서 "Rate limit" 또는 "모델 사용 불가" 메시지
↓
Level 2 (Claude Code)로 전환
```

## 🔄 Level 2: Claude Code (백업)

### 사용 시점
- ⚠️ **Cursor 크레딧 소진 시**
- ⚠️ 무료 API 티어 (일일 한도 ~500 메시지)
- ⚠️ 대화형 디버깅이 필요한 경우

### 크레딧 정보
- **무료 티어**: Anthropic API 프리티어
- **일일 한도**: ~500 메시지 (정확한 한도는 변동)
- **리셋**: 매일 자정 UTC (오전 9시 PST/오전 8시 PDT)
- **속도**: 빠름 (API 호출)

### 사용법
```bash
cd ~/Projects/geekbrox/teams/game/godot/dream-collector
code .

# Cline 패널
@PROJECT_CONTEXT.md
"작업 내용"
```

### 최적화 팁
- **짧은 대화**: 불필요한 대화 최소화
- **명확한 요청**: 한 번에 정확히 전달
- **파일 참조**: @-mention으로 컨텍스트 제공
- **대화 재시작**: 컨텍스트 누적 방지

### 크레딧 추적
```
Cline 우측 하단: "API Usage" 확인
또는
Settings → Cline → Usage Statistics
```

### 크레딧 소진 시
```
Claude Code에서 "Rate limit exceeded" 메시지
↓
Level 3 (Atlas)로 전환
```

## 🔄 Level 3: Atlas (최후 수단 + 관리자)

### 사용 시점
- 🚨 **Cursor + Claude Code 모두 소진 시**
- 🎯 **전체 정리/관리가 필요할 때**
- 🎯 **복잡한 아키텍처 결정**
- 🎯 **Git/Notion 관리**

### 크레딧 정보
- **유료 API**: Anthropic Claude API (Steve 계정)
- **월 한도**: $100 (조정 가능)
- **현재 사용량**: ~$15/월 (2026-02-25)
- **메시지당 비용**: ~$0.01-0.05 (모델/길이에 따라)

### Atlas의 특별 역할

#### 1. 전체 이해 & 정리
**언제**: 코드가 복잡해졌을 때, 프로젝트 방향 전환 시

**작업 예시**:
- Cursor/Claude Code에서 여러 작업 후 전체 리뷰
- 코드 정합성 확인
- 아키텍처 검증
- 문서 업데이트 (PROJECT_CONTEXT.md, CHANGELOG.md)
- 중복 코드 식별

**요청 방법** (텔레그램):
```
"지금까지 Cursor/Claude Code로 작업한 내용 전체 리뷰해줘.
- CHANGELOG.md 보고 변경 사항 이해
- 코드 정합성 확인
- 개선 제안
- 문서 업데이트"
```

#### 2. 공유 & 문서화
**언제**: 작업 완료 후, 팀 공유 필요 시

**작업 예시**:
- CHANGELOG.md 정리
- PROJECT_CONTEXT.md 업데이트
- Notion GDD 업데이트
- 팀원들에게 변경 사항 요약
- 주간 리포트 작성

**요청 방법**:
```
"이번 주 작업 내용을 Notion에 정리해줘:
- 주요 기능 추가
- 버그 수정 목록
- 다음 주 계획"
```

#### 3. 복잡한 작업
**언제**: Cursor/Claude Code가 이해하지 못하는 복잡한 시스템 변경

**작업 예시**:
- 전투 시스템 전체 재설계
- 데이터베이스 구조 변경
- 새로운 핵심 시스템 추가
- 대규모 리팩터링 계획

**요청 방법**:
```
"전투 시스템을 턴제에서 실시간으로 변경하고 싶어.
전체 아키텍처 설계 + 단계별 구현 계획 만들어줘"
```

#### 4. 승인 & 실행
**언제**: 항상 (Git push, Notion update)

**작업**:
- Git push 승인 및 실행
- Notion API 호출
- 팀 알림

### 사용법 (텔레그램)
```
Steve → Atlas:
"[작업 타입] 작업 내용"

예시:
"[정리] 오늘 작업 전체 리뷰 + CHANGELOG 정리"
"[공유] Notion에 주간 리포트 작성"
"[복잡] 전투 시스템 재설계 계획"
"[승인] Git push 해줘"
```

### 최적화 팁
- **일괄 처리**: 여러 질문 한 번에
- **명확한 범위**: 정확히 무엇이 필요한지
- **정기 리뷰**: 주 1-2회 전체 정리
- **긴급만 즉시**: 일상 작업은 Cursor/Claude Code

### 크레딧 리셋 대기
```
Atlas 크레딧도 부족 시:
1. 긴급 작업만 Atlas 사용
2. Cursor/Claude Code 크레딧 리셋 대기 (시간 확인)
3. 리셋 후 Level 1로 복귀
```
