#### 2. **InRun_v4 자동 진행 시스템 구현**

**RunProgressBar 자동 진행 기능:**
```gdscript
const TIME_PER_NODE = 3.0  // 노드당 3초
var is_auto_progressing: bool = false
var progress_to_next: float = 0.0

func _process(delta):
    if not is_auto_progressing or paused:
        return
    
    progress_to_next += delta / TIME_PER_NODE
    update_progress_smooth(progress_to_next)
    
    if progress_to_next >= 1.0:
        _advance_to_next_node()

// 시그널:
signal node_reached(node_index: int, node_data: Dictionary)
signal run_completed()
```

#### 3. **재화 시스템 완전 통합**

**GameManager 시그널 시스템 강화:**
```gdscript
// 모든 재화 변경 시:
1. 시그널 발생 (reveries_changed, gems_changed, energy_changed)
2. 자동 저장 (SaveSystem.save_game())
3. 로그 출력

// 적용된 함수:
func add_gold(amount: int):
    reveries += amount
    reveries_changed.emit(reveries)
    SaveSystem.save_game()  // Auto-save ✅

func spend_gold(amount: int) -> bool:
    if reveries >= amount:
        reveries -= amount
        reveries_changed.emit(reveries)
        SaveSystem.save_game()  // Auto-save ✅
        return true
    return false
```

#### 6. **에너지 표시 문제 수정**

**문제:** 에너지 오브가 항상 0으로 표시
**원인:**
1. EnergyOrb 동적 로드 시 `_ready()` 자동 호출 안 됨
2. 초기 에너지 동기화 누락
3. Null 체크 부족

**해결:**
```gdscript
// CombatBottomUI._ready()
func _ready():
    if energy_orb:
        var EnergyOrbScript = load("res://ui/components/EnergyOrb.gd")
        energy_orb.set_script(EnergyOrbScript)
        energy_orb._ready()  // ✅ 수동 호출
        energy_orb.set_energy(3, 3)

// CombatBottomUI._on_enter()
func _on_enter():
    // ✅ CombatManager에서 현재 에너지 동기화
    if CombatManager.hero and "energy" in CombatManager.hero:
        var current_energy = CombatManager.hero.get("energy", 0)
        _on_energy_changed(current_energy, CombatManager.ENERGY_MAX)
```
