## 📌 v0 사용 전 필수 설정 프롬프트

**모든 화면 작업 시작 전 반드시 이 시스템 프롬프트를 먼저 입력:**

```
시스템 컨텍스트:
모바일 방치형 덱빌딩 게임 UI 제작 중입니다.

기술 스택:
- React + TypeScript + Tailwind CSS + shadcn/ui
- 기준 해상도: 390×844px (세로 모드 전용, 모바일)
- 렌더링: 웹 미리보기이지만 실제 모바일 게임 UI처럼 보여야 함

공통 테마 변수 (CSS Custom Properties로 구현):
:root {
  /* Dream 테마 (꿈 수집가) */
  --color-primary: #7B9EF0;
  --color-secondary: #C4A8E8;
  --color-accent: #F5F0FF;
  --color-bg-main: #0D1B3E;
  --color-bg-panel: rgba(255, 255, 255, 0.10);
  --color-currency-1: #FFE066;
  --color-currency-2: #E8D5FF;
  --radius-card: 16px;
  --radius-button: 20px;
}

레이아웃 원칙:
- 상단: 재화 바 64px + Safe Area
- 중앙: 메인 비주얼 영역 (가변)
- 하단: 메인 액션 버튼 72px + 5탭 내비게이션 80px
- 모든 버튼 최소 44×44px (접근성)
- 글래스모피즘 패널: backdrop-blur + 반투명 배경

이 컨텍스트를 기억하고 이후 요청하는 화면을 제작해주세요.
```


### 코드 품질 확인 사항

v0가 코드를 생성하면 다음을 확인:

```
1. 색상이 하드코딩되어 있으면 → CSS 변수로 교체 요청
   "모든 색상값을 CSS Custom Properties로 리팩터링해주세요"

2. 고정 픽셀 크기가 많으면 → flex/grid 기반으로 요청
   "width: 390px 고정 대신 max-width와 flex를 사용해주세요"

3. 인라인 스타일이 많으면 → Tailwind 클래스로 요청
   "인라인 style 속성 없이 Tailwind 클래스만 사용해주세요"
```

### Figma 이전을 위한 코드 추출 포인트

v0 코드에서 Figma 작업에 필요한 정보 추출:

```typescript
// 1. 색상값 추출 (CSS 변수 또는 직접 색상값)
// → Figma Variables에 입력

// 2. 간격값 추출 (padding, margin, gap)
// → Figma Auto Layout 설정에 반영

// 3. 폰트 크기/굵기 추출
// → Figma Text Style 설정에 반영
```
