## 🎯 핵심 기능

### 1. 카드 탐색
- **3열 그리드:** 카드 111×156px, 간격 12px
- **스크롤:** 무한 스크롤 지원
- **카드 정보:** 이름, 타입, 희귀도, 효과, 에너지 코스트

### 2. 필터링
- **타입 필터:** 전체/수집/액션/시너지/이벤트
- **희귀도 필터:** 전체/일반/언커먼/레어/전설
- **소장 상태:** 전체/보유/미보유/최대 보유

### 3. 검색 및 정렬
- **검색:** 카드 이름 실시간 검색
- **정렬:** 이름순/희귀도순/비용순/최근 획득/즐겨찾기

### 4. 컬렉션 관리
- **진행도 표시:** "34 / 200 (17%)" + 프로그레스 바
- **마일스톤:** 목표 달성 시 보상 알림
- **새 카드 배지:** 최근 획득 카드 강조
- **즐겨찾기:** 별 아이콘 토글

### 5. 카드 상세 보기
- **팝업:** 카드 탭 시 대형 상세 팝업 (340×560px)
- **정보:** 일러스트, 효과, 레벨, 보유 수량
- **액션:** [업그레이드] [덱에 추가] [즐겨찾기]

## 💡 주요 설계 결정

### 1. 3열 그리드 선택 이유
- **가독성:** 카드 정보 충분히 표시
- **효율성:** 한 화면에 6~9장 표시
- **일관성:** 업계 표준 (Hearthstone, Slay the Spire)

### 2. 희귀도 시각화
- **테두리 색상:** 명확한 구분
- **배경 틴트:** 일러스트 영역 색상 강조
- **전설 Glow:** 특별함 시각적 표현

### 3. 필터 우선순위
1. 타입 (가장 많이 사용)
2. 희귀도
3. 소장 상태
4. 검색 (정확한 카드 찾기)

### 4. 모바일 최적화
- **안전 영역:** 상단 44px, 하단 24px
- **터치 영역:** 최소 44×44px
- **햅틱 피드백:** 주요 인터랙션
- **스크롤 성능:** 가상 스크롤 (200장+)

## 📋 기술 사양 요약

### React 컴포넌트 구조

```typescript
<CardLibraryScreen>
  <Header />                    // 뒤로/제목/더보기
  <StatsBar />                  // 컬렉션 진행도
  <FilterBar />                 // 타입 필터
  <SearchBar />                 // 검색 + 정렬
  <CardGrid>                    // 가상 스크롤
    <CardThumbnail />           // 각 카드 (×200)
  </CardGrid>
  <BottomNav />                 // 5탭
  <CardDetailPopup />           // 조건부 렌더링
</CardLibraryScreen>
```

### 데이터 모델

```typescript
interface Card {
  id: string;
  name: string;
  type: 'collection' | 'action' | 'synergy' | 'event';
  rarity: 'common' | 'uncommon' | 'rare' | 'legendary';
  cost: number;
  effect: string;
  illustration: string;
  level: number;
  owned: number;
  isNew: boolean;
  isFavorite: boolean;
}
```

### 성능 최적화
- **가상 스크롤:** react-window (200장 대응)
- **이미지 레이지 로딩:** Intersection Observer
- **필터 디바운스:** 300ms
- **CSS 애니메이션:** transform/opacity만 사용
