## 📊 재화 시스템 v2.0

### **변경 전 (v1.0)**
```
Reveries (꿈 조각) - 단일 재화
  ↓
게임 아이템 직접 구매
```

### **변경 후 (v2.0)**
```
현금 ($)
  ↓
💎 Gems (보석) - 유료 화폐
  ↓
🪙 Gold (골드) - 게임 재화
⚡ Energy (에너지) - 전투 재화
```

### **주요 재화**
| 재화 | 이전 | 현재 | 획득 방법 |
|------|------|------|-----------|
| 기본 재화 | Reveries | 🪙 Gold | Idle 수집, 전투, 교환 |
| 유료 화폐 | (없음) | 💎 Gems | IAP 결제, 이벤트 |
| 전투 재화 | (없음) | ⚡ Energy | 자동 회복, 교환 |

## 🪟 AlertModal 컴포넌트

### **기능**
- 재사용 가능한 팝업/모달 컴포넌트
- 다양한 알림/확인 다이얼로그

### **용도**
```gdscript
# 간단한 알림
alert_modal.show_info("알림", "메시지")

# 경고
alert_modal.show_warning("경고", "메시지")

# 확인 다이얼로그
alert_modal.show_confirm("제목", "정말 삭제하시겠습니까?", "삭제", "취소")

# 재화 부족 (자동 포맷)
alert_modal.show_insufficient_currency("보석", 100, 50)
```

### **UI 특징**
- ✅ 배경 50% 어둡게
- ✅ CanvasLayer (최상단 렌더링)
- ✅ 버튼 1~2개 지원
- ✅ 버튼 색상 커스터마이징
- ✅ 오버레이 클릭으로 닫기

## 🧪 테스트 가이드

### **치트 코드 (개발용)**
MainLobby 화면에서:
- **M키**: Reveries (Gold) +1,000
- **N키**: Reveries (Gold) +10,000
- **G키**: Gems +100 ⭐ (신규)
- **H키**: Gems +1,000 ⭐ (신규)

### **테스트 시나리오**
1. **보석 구매 (IAP 시뮬레이션)**
   - Shop → 💎 보석 구매 탭
   - 아무 패키지 클릭
   - 모달: "구매 완료" + 보석 지급
   - 상단 💎 카운터 증가 확인

2. **재화 교환 (성공)**
   - G키로 Gems 충전
   - Shop → 🪙 재화 교환 탭
   - "골드 1,000" 교환
   - 모달: "교환 완료"
   - 💎 감소, 🪙 증가 확인

3. **재화 교환 (실패)**
   - Gems를 모두 소진

   - 모달: "보석 부족" + [💰 재충전] 버튼

## 📊 밸런싱 지표

### **Idle 수집**
```
1시간 Idle = 10 Gold (기본)
💎 1 Gem ≈ 10 Gold (교환 시)
```

### **에너지 소비**
```
1 Run = 10 Energy
자연 회복: 1 Energy/10분 (6/hour)
1일 자연 회복: 144 Energy (14 Run)
```

### **가격 예시**
```
카드 강화: 100 ~ 1,000 Gold
카드 구매: 500 ~ 5,000 Gold
업그레이드: 1,000 ~ 50,000 Gold
```
